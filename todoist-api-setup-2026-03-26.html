Part 2 — Step-by-Step: Getting Your API Token

Five simple steps to copy your API token from Todoist.

1
Open Todoist in your browser
Go to todoist.com and make sure you're logged in as varunm24@gmail.com.
2
Open Settings
Click your profile picture or avatar in the top-right corner, then click Settings from the dropdown.
3
Go to Integrations → Developer
In the left sidebar of Settings, click Integrations. Then click the Developer tab at the top of that page.
🔗 app.todoist.com/app/settings/integrations/developer
4
Find "API token"
You'll see a section labeled API token. Your personal token is shown there — a long string of about 40 characters.
5
Click "Copy to clipboard"
Click the copy button next to the token. It's now ready to paste anywhere you need it. Treat it like a password — don't share it publicly.

Part 5 — Setting Up the Environment Variable (If Needed)

Only do this if you want to use your Todoist API key in Claude Code or other tools outside of Perplexity Computer. This is a one-time setup.

On Mac (Terminal)

Open Terminal and run this command, replacing YOUR_TOKEN_HERE with the token you copied from Todoist:

# Add to your shell profile (runs once)
echo 'export TODOIST_API_TOKEN="YOUR_TOKEN_HERE"' >> ~/.zshrc
source ~/.zshrc

# Verify it works
echo $TODOIST_API_TOKEN

On Windows (PowerShell)

# Set for current session
$env:TODOIST_API_TOKEN = "YOUR_TOKEN_HERE"

# Set permanently (persists after restart)
[System.Environment]::SetEnvironmentVariable("TODOIST_API_TOKEN", "YOUR_TOKEN_HERE", "User")
ℹ How Claude Code uses it

Once set, Claude Code can access your token automatically when writing scripts that call the Todoist API. You never have to paste the token into code — Claude just references os.environ["TODOIST_API_TOKEN"] or process.env.TODOIST_API_TOKEN.
