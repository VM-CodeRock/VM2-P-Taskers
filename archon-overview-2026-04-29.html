Archon: AI Coding Harness Builder
What it is, how it works, and whether Varun should use it

Open Source MCP Server AI Coding RAG / Knowledge Base Harness Engineering Docker Beta

1. Executive Summary

Plain-English Answer

Archon is a self-hosted command center that sits between you and your AI coding assistant (Claude Code, Cursor, Windsurf, etc.) and gives it a custom knowledge base plus a structured task manager — all exposed through the Model Context Protocol (MCP) so any AI tool can query it. Think of it as a private "brain + to-do list" for your coding agents: you feed it your project docs and define repeatable coding workflows as YAML files, and your AI agents follow that structure instead of winging it every time.

Archon is built by Cole Medin, a well-known AI engineering YouTuber, and is currently in beta with 13,600+ GitHub stars and 31 contributors. The project has evolved from an "AI agent that builds other agents" into a broader platform for what Medin calls harness engineering — the next evolution after prompt engineering and context engineering.

2. The Problem It Solves

AI coding assistants (Claude Code, Cursor, Cline, etc.) are powerful but inconsistent. Every time you start a new conversation or project, the agent starts cold — no memory of your coding standards, no knowledge of your preferred libraries' docs, no consistent workflow. You end up re-explaining context, manually shepherding the agent through the same steps, and getting different results each run.

There are two distinct pain points Archon addresses:

Problem A — Knowledge Gap

Your AI coding assistant doesn't know your custom docs, internal frameworks, or the specific library versions you're using. You're either pasting documentation into every chat window or accepting that the model will hallucinate outdated APIs.

Problem B — Workflow Inconsistency

Multi-step development processes (plan → implement → test → review → PR) require you to manually advance the agent through each stage. There's no enforcement: the agent might skip tests, forget to create a branch, or produce a different output structure than yesterday's run.

The Harness Insight

A harness is the layer on top of the coding agent — the structure that makes AI coding deterministic and repeatable. Just as Dockerfiles standardize infrastructure and GitHub Actions standardize CI/CD, Archon YAML workflows standardize the AI development lifecycle. The AI only runs where it adds value; bash nodes handle the deterministic parts; loop nodes handle retry logic.

3. How It Works — Architecture

Archon runs as a set of Docker microservices on your local machine (or a server). Each service has a distinct role:

Service Port What It Does
Web UI (archon-ui) 3737 Browser dashboard — manage knowledge, projects, tasks, settings, MCP config
API Server (archon-server) 8181 Core business logic: web crawling, document processing, ML/AI ops, Socket.IO real-time updates
MCP Server (archon-mcp) 8051 The interface your AI coding assistant actually connects to — exposes RAG search, task management, and project tools over Model Context Protocol
Agents Service (archon-agents) 8052 PydanticAI agent hosting: document agents, RAG reranking, streaming responses
Agent Work Orders (optional) 8053 Workflow execution engine — runs YAML harness workflows via Claude Code CLI, manages git worktrees

All services back against Supabase (PostgreSQL + PGVector) for knowledge storage and vector search. You can use Supabase's free cloud tier or run it locally.

A. Knowledge Base (RAG)

Feed Archon documentation via web crawler (paste any URL — it detects sitemaps and crawls the whole site) or by uploading PDFs, Word docs, and Markdown files. Archon chunks and embeds them using your choice of OpenAI, Google Gemini, or Ollama (local). When your coding agent needs context, it calls the MCP search_knowledge tool and gets semantically-relevant chunks back, with hybrid search + reranking for precision.

B. MCP Integration

Your AI coding assistant (Claude Code, Cursor, Windsurf, Kiro, etc.) connects to Archon's MCP server as a tool server. The coding assistant can then call tools like search_knowledge, create_task, update_task_status, and get_project_context — using Archon as a live external brain during every coding session. No extra config in each conversation; the connection is persistent.

C. Project & Task Management

Projects in Archon are hierarchical: Project → Features → Tasks. Your AI coding assistant can create, update, and move tasks through statuses in real-time (websocket updates visible in the UI). This gives you a live view of what the agent is working on and what's done, without tab-switching.

D. Harness Workflows (YAML)

The newest and most powerful feature. You define a directed acyclic graph (DAG) of nodes in a YAML file checked into your repo:

name: bug-fix
nodes:
  plan:
    type: ai
    prompt: "Analyze the bug report and create a fix plan"
  implement:
    type: ai
    prompt: "Implement the fix according to the plan"
    depends_on: [plan]
  test:
    type: bash
    command: "npm test"
    depends_on: [implement]
  review:
    type: ai
    prompt: "Review implementation against the bug report"
    depends_on: [test]
  pr:
    type: bash
    command: "gh pr create --title 'Fix: {bug_id}'"
    depends_on: [review]

Four node types: AI (calls Claude Code / Codex), Bash (deterministic shell commands), Loop (iterate until a condition passes), and Approval Gate (pause for human confirmation). Workflows run from CLI, Slack, GitHub webhook, or the Web UI. Each run gets its own git worktree for isolation.

4. How to Use It — Setup

  • 1
    Prerequisites: Install Docker Desktop. Create a free Supabase account. Have an OpenAI API key ready (Gemini and Ollama also supported).
  • 2
    Clone the stable branch:
    git clone -b stable https://github.com/coleam00/archon.git && cd archon
  • 3
    Configure environment:
    cp .env.example .env — then edit .env and add your Supabase URL and service key (use the legacy longer key).
  • 4
    Initialize the database: In your Supabase project → SQL Editor, paste and run the contents of migration/complete_setup.sql.
  • 5
    Start Archon:
    docker compose up --build -d
    This spins up the UI (port 3737), API server (8181), and MCP server (8051).
  • 6
    Complete onboarding: Open localhost:3737. The onboarding flow walks you through selecting your LLM provider and setting your API key.
  • 7
    Ingest your knowledge base: Go to Knowledge Base → "Crawl Website" and paste a docs URL (e.g., your internal wiki, library docs). Upload any PDFs you want the agent to reference.
  • 8
    Connect your AI coding assistant: Go to MCP Dashboard in the UI → copy the MCP config snippet for Claude Code or Cursor. Paste it into your .claude/mcp.json or Cursor MCP settings. Your coding assistant can now call Archon tools.
  • 9
    Create your first project: Projects → New Project → add features and tasks. Your AI coding assistant will see these when you ask it to "use Archon to plan this feature."
Quick Validation Test

After setup, ask Claude Code (with Archon MCP connected): "Search the Archon knowledge base for [your topic] and create a task for implementing it." — If you see the task appear live in the Archon UI, the integration is working correctly.

Tool Category What It Does Well Archon Relationship
Claude Code AI Coding Agent Agentic coding, terminal integration, strong at complex tasks Complementary — Archon MCP plugs directly into Claude Code, giving it knowledge + task management
Cursor AI Code Editor Inline suggestions, codebase-aware Q&A, chat interface Complementary — Archon MCP works with Cursor; adds cross-project knowledge and structured workflows
Cline VS Code Extension Autonomous coding agent, tool use (browser, terminal), MCP support Complementary — Cline can connect to Archon via MCP; Archon provides context Cline lacks natively
Aider CLI Coding Agent Git-native workflow, pair programming feel, open source Overlapping — Both work from the terminal; Archon's harness can orchestrate Aider-like runs but requires more setup
Continue VS Code Extension Open-source Copilot alternative, configurable models Parallel — Continue is an inline editor; Archon is a backend knowledge/task layer; non-competing
RooCode VS Code Extension Multi-agent orchestration, deep filesystem access Partial Overlap — Both do agent orchestration; Archon goes deeper on knowledge management and YAML workflow definition
Windsurf AI Code Editor Cascade agentic flow, contextual code understanding Complementary — Windsurf can connect to Archon MCP; Archon's knowledge layer persists across Windsurf sessions
Key Differentiator

Most AI coding tools are agents — they take instructions and execute. Archon is a harness — it defines the structure those agents operate within. The distinction matters: agents improvise, harnesses enforce. Archon is the only open-source tool in this specific category (as of April 2026).

6. Should Varun Use This?

Context: Your Current Setup

You're running 14 cron-based AI workflows in Perplexity Computer — a sophisticated agentic system with structured task pipelines, Todoist integration, git commit/push automation, Dropbox sync, and portal publishing. Computer already handles orchestration, state management, and multi-step workflow execution. This context matters significantly for the assessment below.

Where Archon Adds Real Value

  • You're writing code in Claude Code / Cursor and want persistent, searchable access to your own docs (internal Changeis frameworks, custom methodologies)
  • You're doing active software development with repeatable workflows (not just one-off scripts)
  • You want to encode your development process as version-controlled YAML that runs identically every time
  • Your coding agent keeps forgetting context between sessions
  • You need parallel agent execution with git worktree isolation for complex projects

Where Archon Duplicates or Conflicts

  • Your orchestration layer (Perplexity Computer + cron) already handles structured workflows — Archon's harness is a parallel system, not a replacement
  • Archon requires self-hosting (Docker, Supabase) — adds infrastructure to manage
  • It's in beta — the README explicitly warns of incomplete functionality
  • If your AI work is primarily in Computer (not local coding sessions), the MCP integration doesn't apply
  • Task management in Todoist (already integrated with VM2) covers the project/task layer Archon provides

Honest Assessment

The value proposition is real, but the fit is narrow given your current setup.

Archon shines for software engineers who do heavy, iterative local coding with Claude Code or Cursor and want persistent knowledge + structured workflows. Your primary AI workflow (Computer + cron) already provides orchestration, task management, and structured outputs — Archon would be a second system with overlapping responsibilities.

The one clear gap Archon could fill: If you're using Claude Code locally for Changeis-specific software development (not VM2 task pipelines), Archon's knowledge base — fed with Changeis frameworks, proposal templates, and methodology docs — could meaningfully improve Claude Code's outputs by giving it company-specific context that persists across sessions. That's the specific, non-duplicative use case worth exploring.
