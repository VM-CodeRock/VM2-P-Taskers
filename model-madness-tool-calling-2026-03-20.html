Overview

Key Takeaway

Scaffolding matters as much as model capability. Alephic's March Madness bracket challenge reveals that workflow design, validation tools, and guided task decomposition can close the performance gap between frontier and budget AI models — a critical insight for anyone building production agentic systems.

Alephic, an AI strategy and software firm, built a March Madness–style bracket tournament to benchmark how well 45 AI models wield tools in a real-world reasoning task. Rather than testing models on synthetic benchmarks, Model Madness asks each model to research NCAA teams, reason about matchups, and submit a structured 63-game bracket — using web search, page fetching, a browser, and a calculator.

The tournament is live at model-madness.alephic.ai and was still in progress (Round of 64, 16 games decided) as of this writing. The results so far surface sharp lessons about the engineering requirements for multi-model agentic systems.

The Experiment

Each of the 45 models receives three entries — one at each difficulty level (Easy, Medium, Hard) — for a total of 100 bracket submissions. Models must use the provided tool set to research NCAA tournament teams, evaluate matchups, and produce a valid 63-pick bracket. The bracket is validated against real tournament structure: correct pick counts per round, valid team names, and carry-forward constraints (eliminated teams cannot reappear).

Why It Matters

This is not a trivia quiz or a synthetic benchmark. It tests the full agentic loop: information retrieval, multi-step reasoning, structured output generation, and tool orchestration — all against a verifiable real-world outcome. The difficulty levels isolate which failures are caused by model reasoning versus poor tool scaffolding.

Methodology — Three Difficulty Levels

The three tiers control how much scaffolding the model receives. This design separates raw reasoning ability from the engineering quality of the harness around the model.

Level Approach Additional Tools What It Tests
Hard Submit full 63-game bracket in a single shot. Formatting errors and hallucinated team names penalized as wrong picks. None (research tools only) Reasoning + recall + structured output simultaneously
Medium Full bracket in one shot, but with lookup & validation helpers. Score reflects prediction quality, not data-entry luck. lookup_team, lookup_game, validate_bracket Prediction quality with formatting safety net
Easy Predict one round at a time. Valid team pool shrinks each round (64→32→16→8→4→2). Nearly impossible to produce formatting errors. All medium tools + round-by-round guided flow Pure prediction quality, isolated from bracket construction

Tool Set

All models at every difficulty level have access to the core research tools. Medium-level entries gain three additional helper tools.

Tool Available At Description
web_search All levels Search the web via Firecrawl. Returns top 5 results with titles and snippets.
web_fetch All levels Fetch a page as markdown, truncated to 4,000 characters.
use_browser All levels Real browser with 5-second JS rendering wait. Fallback for JavaScript-heavy pages.
calculator All levels Safe math evaluator supporting +, −, ×, ÷, %, and parentheses.
submit_bracket All levels Submit final 63-pick bracket. Runs validate_bracket internally.
lookup_team Medium+ Find a tournament team by partial name, slug, seed, region, or display name.
lookup_game Medium+ Takes a game ID; returns both teams, seeds, and regions.
validate_bracket Medium+ Pre-submission check: correct pick counts (32-16-8-4-2-1), valid game IDs, team presence, carry-forward constraints.

Tech Stack

The platform runs on NextJS (Vercel), using the Vercel AI SDK for model calls, Vercel WorkflowDevKit for async processing, and Vercel AI gateway for unified access to all 45 models through a single provider. Firecrawl powers web search and page fetching.

Key Findings on Tool Calling & Scaffolding

Finding 1 — The Scaffolding Gap Is Real

Less capable models (GPT-4o mini, Gemini Flash) require significantly more scaffolding to achieve quality comparable to frontier models. Alephic iterated through multiple scaffolding improvements — teams-as-enum constraints, validators, submission-as-tool — to close the gap. Even case-sensitivity bugs (e.g., bradley braves vs. bradley-braves) caused failures until caught and fixed.

Finding 2 — Cross-Model Engineering Is a Distinct Discipline

Daily use of top-tier models (Opus, GPT-5.4, Gemini Pro 3.1) masks the difficulties that cheaper models face. Building for multiple model tiers simultaneously is fundamentally different from single-model engineering. Production agentic systems must maintain multiple scaffolding regimes — not one-size-fits-all prompts.

Finding 3 — Workflow Design Eliminates Entire Error Classes

The Easy mode — round-by-round prediction with a shrinking valid-team pool — makes formatting errors nearly impossible. This proves that task decomposition and constrained workflows can neutralize model weaknesses without improving the model itself. The harness is the product.

Finding 4 — The Gap Between Model Classes Persists

Despite extraordinary progress in budget models, the capability gap between frontier and lower-tier models appears permanent. Success at scale requires tooling and scaffolding tailored to each capability tier — not a single engineering approach applied uniformly.

Implications for Agentic AI

Bottom Line

For production agentic systems, you cannot simply pick the best model and ship. You need scaffolding that degrades gracefully across capability tiers — and you need to test against real-world tasks, not synthetic benchmarks.

  • Invest in scaffolding, not just models. Validators, guided workflows, and structured output constraints deliver outsized returns — especially when supporting multiple model tiers.
  • Design for the weakest model you'll ship. If your agentic system must fall back to cheaper models for cost or latency reasons, the scaffolding must compensate for reduced reasoning ability.
  • Task decomposition is a force multiplier. Breaking a complex 63-game bracket into round-by-round steps with shrinking option sets nearly eliminated errors. Apply this pattern to any multi-step agentic workflow.
  • Real-world benchmarks reveal what synthetic ones miss. Model Madness tests retrieval, reasoning, structured output, and tool orchestration simultaneously — a more honest signal of agentic readiness than isolated capability tests.
  • Cross-model testing is non-optional. A system that works on GPT-5.4 may fail silently on GPT-4o mini. Agentic platforms must be validated across the full model spectrum they intend to support.
