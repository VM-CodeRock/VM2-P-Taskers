1 · What These 89 Commands Actually Are
Critical Framing — Read This First

Most of these are not real slash commands. Claude.ai (web and mobile) has essentially no native slash-command system. Claude Code (the CLI tool) has a small set of actual built-in commands, and users can define their own custom commands. The viral infographic you received is a prompt-pattern cheat-sheet dressed up to look like a hotkey reference.

This matters because it changes how you use them: you don't type /summarize into a command bar and hit enter. You start your message with "summarize this" or "give me the key points from" and Claude responds correctly. The value is as a mental checklist of what Claude can do, not as keyboard shortcuts.

Context What's Real What Isn't
Claude.ai web/mobile None of the 89; Projects with custom instructions; file uploads All 89 slash commands as interactive hotkeys
Claude Code (CLI) /help, /clear, /compact, /memory, /init, /agents, /model, /status — plus any custom .claude/commands/*.md files you create The other ~81 commands
Anthropic API System prompts that can encode any of these behaviors; extended thinking mode; structured outputs Slash commands as a protocol
Claude Projects Custom instructions (effectively /tone, /style, /context, /constraints baked in permanently); knowledge files Runtime slash commands
The Useful Reframe

Think of the infographic not as a command reference but as a vocabulary list for directing Claude. The commands name behaviors Claude understands natively from plain English. Use them as a pre-flight checklist: before sending a long prompt, ask yourself — do I need to /focus the scope, /constrain the output length, or /define a term first? That mental habit is the actual value here.

3 · Where Slash Commands ARE Real — Power User Setup

The one place slash commands work exactly as the infographic implies is Claude Code with custom command files. Here is how to set this up for Changeis federal BD work.

How Claude Code Custom Commands Work

In any project directory, create a folder: .claude/commands/. Each .md file in that folder becomes a slash command available in your Claude Code session. These are templated prompts — you write the instruction once, invoke it repeatedly.

Real Example: .claude/commands/changeis-deep-dive.md
# .claude/commands/changeis-deep-dive.md # Invoke with: /changeis-deep-dive [opportunity name or SAM notice ID] You are a federal BD analyst for Changeis, Inc. Changeis is a management consulting and IT firm focused on DHS, DOT, and FAA. Varun Malhotra is President. The firm is 8(a)-eligible and competes on STARS III, OASIS+, and agency-specific IDIQs. For the opportunity provided, produce a structured deep-dive covering: 1. OPPORTUNITY SUMMARY - Agency / Component / Program Office - Contract vehicle (if known) - Estimated value and period of performance - NAICS code and small business set-aside status - Key dates (RFI, draft RFP, final RFP, award) 2. STRATEGIC FIT ANALYSIS - Alignment with Changeis core capabilities (rate 1-5 with rationale) - Incumbent (if known) and re-compete risk - Teaming requirements or likely teaming angles - PWIN estimate with key assumptions 3. BID/NO-BID RECOMMENDATION - Recommendation: Pursue / Watch / Pass - Top 3 deciding factors - Required actions if pursuing (next 2 weeks) 4. INTELLIGENCE GAPS - What do we need to know that we don't? - Suggested capture activities Format as a structured brief. Be direct. Flag uncertainty explicitly.

Claude Projects — The No-Code Equivalent

If you don't want to use Claude Code CLI, Claude Projects achieves similar results. Create a Project per major pursuit or business area, add custom instructions encoding your tone/style/constraints, and upload knowledge files (RFPs, cap statements, org charts). This is the zero-friction path to the same outcome.

4 · Top 10 Highest-Leverage Prompt Patterns for Varun's Daily Work
1

/pros-cons + /recommend — Bid/No-Bid Triage

pros-cons + recommend

Paste any SAM.gov synopsis and say: "Pros and cons of pursuing this as a prime for Changeis (8a, DHS-focused, ~50-person firm). Then give me your recommendation with the 3 deciding factors." Cuts PWIN debate from 45 minutes to 5. Use daily during SAM monitoring.

2

/research + /sources — Opportunity Diligence

research + sources

Before any capture call: "Research [DHS component/program office]. Give me: current budget cycle priorities, active or upcoming IT/management consulting contracts, recent news, and the key contracting officers. Cite sources." 10 minutes of research that used to take 2 hours.

3

/summarize + /key-points — OIG/GAO Report Triage

summarize + key-points

Paste OIG or GAO report text (or upload the PDF): "Summarize this report. Key points only — 5 bullets max. Flag any findings that implicate IT services contractors or create new consulting opportunities at [agency]." The VM2 daily brief surfaces these; this pattern extracts BD signal from them.

4

/rewrite — Proposal Section Elevation

rewrite

"Rewrite this technical approach section to lead with quantified outcomes, not activities. Audience: DHS CISA source selection board. Evaluation criterion: Technical approach (40%). Word limit: 500. Remove all passive voice." Takes a mediocre draft to competitive in one pass.

5

/analyze + /compare — Teaming Partner Evaluation

analyze + compare

"I'm evaluating two potential teaming partners for a DHS opportunity. Partner A: [details]. Partner B: [details]. Analyze both against these criteria: past performance relevance, technical depth, contract vehicle access, small business status. Then compare and tell me which strengthens our proposal more."

6

/table — Pipeline Visualization

table

After getting opportunity data from VM2 or SAM monitoring: "Convert this list of opportunities into a table. Columns: Agency, Title, Due Date, Est. Value, Vehicle, Set-Aside, Changeis Fit. Rate fit High/Med/Low based on our DHS IT consulting focus." Instant visual pipeline triage.

7

/checklist — Pre-Submission Compliance QA

checklist

"Generate a pre-submission compliance checklist for this RFP. Include: required volumes and page limits, mandatory attachments (reps & certs, past performance references, team org chart), formatting requirements per Section L, and common disqualifying errors." Zero-cost QA before every submission.

8

/tldr — Executive Brief from Long Documents

tldr

For any document over 10 pages (budget justifications, contract award notices, strategic plans): "TL;DR in 5 bullets. Focus on: budget numbers, new program starts, contract vehicles mentioned, and anything relevant to management consulting or IT services." Your reading time drops by 80%.

9

/feedback + /improve — BD Outreach Self-Review

feedback + improve

Before sending any BD email to a CO or program office: "Review this outreach email. Flag: anything that sounds sales-y, vague, or non-specific. Then rewrite the opening paragraph and subject line to be more relevant to a GS-14 program manager at DHS." Eliminates the most common BD email mistakes.

10

/brainstorm — Capture Strategy Generation

brainstorm

When entering a new pursuit: "Brainstorm 10 ways Changeis could differentiate on this DHS opportunity. Focus on: unique past performance angles, potential teaming structures, technical differentiators, and price-to-win positioning. Be creative — include unconventional approaches." Breaks capture planning out of groupthink.

5 · Beyond the Infographic — 5 Things More Useful for Federal BD

The infographic lists prompt patterns. Here are five actual Claude capabilities that are more transformative for your specific work — and that most users don't know exist or underuse.

Feature 1

Extended Thinking Mode

Claude claude-3-7-sonnet has a "thinking" mode that lets it reason step-by-step before answering. For complex bid/no-bid decisions with many variables, or investment thesis evaluation, this produces materially better analysis than standard mode. Access via the API or Claude.ai's extended thinking toggle. Especially valuable for anything resembling a multi-criteria decision problem.

Feature 2

Claude Projects with Knowledge Files

This is the highest-ROI feature for your BD workflow. Create one Project per major pursuit or agency. Upload: the RFP/SOW, Changeis cap statement, relevant past performance write-ups, org chart, and any intel on the program office. Every conversation in that Project has this context automatically — no re-pasting. A dedicated "DHS CISA" Project with 5 uploaded files turns every conversation into an expert-informed analysis.

Feature 3

MCP (Model Context Protocol) Integrations

Claude Code supports MCP servers — connectors that give Claude real-time access to external tools: SAM.gov, USASpending.gov, Perplexity search, Google Drive, and more. The VM2 system already uses this architecture. You can set up a local Claude Code workspace where /changeis-deep-dive pulls live SAM data automatically, rather than requiring you to paste it. This is the path from "prompt patterns" to genuine automation.

Feature 4

System Prompts (API) for Repeatable Workflows

If you use Claude via API (or tools like the VM2 system), system prompts let you encode your entire persona, constraints, and knowledge base permanently — not re-specified each session. A well-crafted Changeis system prompt that includes your capabilities, current pursuits, tone preferences, and BD framework turns every API call into a contextualized analyst response. This is what powers VM2's daily briefs.

Feature 5

Claude for YPO & Investment Work

The infographic focuses on productivity commands but misses Claude's value as a thinking partner for high-stakes decisions. For YPO forum prep: upload a speaker's bio + recent writing, ask Claude to generate 5 substantive questions and predict likely positions. For investment review (Aadyah, angel): paste a pitch deck and ask for bear/bull cases, comp analysis, and the 3 questions you must get answered before committing capital. These use cases have no "command" — they're open-ended analytical conversations.
