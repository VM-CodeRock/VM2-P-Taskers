AWS Positioning — What Role Does AWS See for Itself in ATLAS?

Assessment: AWS as Platform Partner / ISV-of-Record

The package positions AWS as the technology platform of record — not as a prime or co-prime. AWS explicitly states "together with our AWS Partners" throughout, framing the relationship as: AWS provides the platform infrastructure, AI tooling, and funding programs; the integrator (prime + subs) delivers the modernization factory, rationalization strategy, and program management. This is a platform-partner model, analogous to how IBM positioned itself via BAH. AWS is not competing against Changeis — it is competing to be the cloud infrastructure layer chosen over Azure GovCloud or Google Cloud.

What AWS Explicitly Offers Partners

  • MAP Funding: AWS Migration Acceleration Program provides financial investment across Assess, Mobilize, and Migrate/Modernize phases — can materially offset factory standup costs in the ATLAS cost model
  • GSA OneGov Credits: Up to $1B in federal civilian savings (through Dec 31, 2028) on core AWS services — directly reduces hosting CLIN (X006) cost structure
  • Technical Deep Dive: Katharine Day explicitly offered a technical deep-dive session — this is an invitation to co-develop the solutioning approach
  • ProServe Embedded Support: AWS ProServe SA/architects can embed directly with the delivery team (§Conclusion references "AWS ProServe…delivering outcomes, not just licenses")
  • Proven FAA Foundation: AWS claims multiple FAA workloads already live — minimizing transition risk in the Phase 2 migration sequencing narrative

What the Package Does NOT Address (Gaps to Probe)

  • No LOI/MDF requirement stated: Package does not specify whether an LOI, Memorandum of Understanding, or Market Development Funds commitment is required to access MAP funding — must ask
  • Partner ecosystem: open or curated? The package does not state whether ATLAS partner slots are first-come-first-served or curated by AWS (i.e., can all primes access MAP, or does AWS pick preferred integrators?)
  • No work share / teaming structure terms: AWS is not proposing a traditional TA — the terms of access to MAP, ProServe, and technical resources are not documented; this must be negotiated
  • No exclusivity language: AWS will almost certainly provide this same package to all 5 ATLAS Phase 1 finalists; Changeis/BAH must differentiate on how deeply they leverage the AWS platform
  • No cliff analogous to BAH OTA: No hard deadline in the package for joining the AWS partner ecosystem — but delaying intro call while competitors move forward is a strategic risk
