Method 1

launchd — The macOS Native Way ✓ Recommended

launchd is macOS's built-in job scheduler. It runs even after reboots, handles logging automatically, and is far more reliable than crontab on modern Macs. This is the method you should use.

Step 1 — Create the plist file

Open Terminal (press ⌘ Space, type "Terminal", hit Enter). Then run this command to create the file in the right place:

Terminal — paste and run this 
nano ~/Library/LaunchAgents/com.changeis.crm-daily.plist

Step 2 — Paste the plist content

The nano text editor opens. Copy the entire block below, then paste it with ⌘V:

com.changeis.crm-daily.plist — copy everything below 
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>

    <!-- Unique identifier for this job -->
    <key>Label</key>
    <string>com.changeis.crm-daily</string>

    <!-- The command to run -->
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>/Users/varunmalhotra/CHANGEIS Dropbox/Varun Malhotra/_Leapfrog/Corp_CRM Management/config/run-crm-daily.sh</string>
    </array>

    <!-- Run daily at 8:00 AM -->
    <key>StartCalendarInterval</key>
    <dict>
        <key>Hour</key>
        <integer>8</integer>
        <key>Minute</key>
        <integer>0</integer>
    </dict>

    <!-- Log output to a file -->
    <key>StandardOutPath</key>
    <string>/Users/varunmalhotra/Library/Logs/crm-daily.log</string>

    <!-- Log errors to a separate file -->
    <key>StandardErrorPath</key>
    <string>/Users/varunmalhotra/Library/Logs/crm-daily-error.log</string>

    <!-- Run even if the time was missed (e.g. Mac was asleep) -->
    <key>RunAtLoad</key>
    <false/>

</dict>
</plist>
To change the run time
Change <integer>8</integer> (Hour) to any hour in 24-hour format. For example, 9 = 9am, 18 = 6pm, 7 = 7am.

Step 3 — Save and exit nano

1

Press Ctrl + X

This starts the exit process.

2

Press Y to save

It asks "Save modified buffer?" — press Y.

3

Press Enter to confirm filename

The file is saved and nano closes.

Step 4 — Register the schedule with macOS

Run this command in Terminal to activate the schedule:

Terminal 
launchctl load ~/Library/LaunchAgents/com.changeis.crm-daily.plist

Step 5 — Verify it's loaded

Terminal 
launchctl list | grep changeis

You should see a line with com.changeis.crm-daily. If the first column shows a number other than -, that's the last exit code (0 = success).

Optional — Test it right now (without waiting)

Terminal — runs the script immediately 
launchctl start com.changeis.crm-daily

To stop or remove the schedule

Terminal — unload (stops the schedule) 
launchctl unload ~/Library/LaunchAgents/com.changeis.crm-daily.plist
Important: Mac must be on and logged in
launchd runs under your user account, so your Mac must be awake and you must be logged in at the scheduled time. If the Mac is asleep, the job will run the next time it wakes up (if you set RunAtLoad to true) or wait until the next scheduled time.
Method 2

crontab — The Simple Way Simple

crontab is a Unix-standard scheduler that works on any Mac. It's easier to set up but gives you less control over logging and reliability on modern macOS compared to launchd.

Step 1 — Open crontab editor

Terminal 
crontab -e

This opens a text editor (usually vim or nano, depending on your Mac settings).

Step 2 — Add the schedule line

Add this line to run every day at 8:00 AM:

crontab entry — paste this line 
0 8 * * * /bin/bash '/Users/varunmalhotra/CHANGEIS Dropbox/Varun Malhotra/_Leapfrog/Corp_CRM Management/config/run-crm-daily.sh' >> /Users/varunmalhotra/Library/Logs/crm-daily.log 2>&1
Cron time format: minute hour day month weekday
  • 0 8 * * * = 8:00 AM every day
  • 0 9 * * * = 9:00 AM every day
  • 0 7 * * 1-5 = 7:00 AM weekdays only
  • 30 8 * * * = 8:30 AM every day

Step 3 — Save and exit

If the editor is vim: press Esc, then type :wq and press Enter.

If the editor is nano: press Ctrl+X, then Y, then Enter.

macOS may ask for Full Disk Access
On macOS Catalina or later, cron may need Full Disk Access. Go to System Settings → Privacy & Security → Full Disk Access and add /usr/sbin/cron.
Method 3

Automator + Calendar — The Visual Way Visual

If you prefer a graphical approach with no Terminal commands, Automator and the Calendar app can run your script on a schedule.

1

Open Automator

Press ⌘ Space, type "Automator", press Enter. When it asks "Choose a type", select Calendar Alarm.

2

Add a "Run Shell Script" action

In the search box on the left, type "Run Shell Script". Double-click it to add it to your workflow.

3

Paste the command

In the script box, paste:

bash '/Users/varunmalhotra/CHANGEIS Dropbox/Varun Malhotra/_Leapfrog/Corp_CRM Management/config/run-crm-daily.sh'
4

Save the Automator workflow

Click File → Save. Name it something like "CRM Daily Run". This automatically adds it to your Calendar app.

5

Set the recurring schedule in Calendar

Open Calendar. Find the new event called "CRM Daily Run". Double-click it → click Edit → set the time to 8:00 AM → set Repeat to Every Day → click Done.

Automator limitation
Calendar-based Automator alarms only fire if the Calendar app is running. For a more reliable schedule, use Method 1 (launchd).
Logs & Troubleshooting

Checking Logs & Fixing Problems

View the output log

Terminal — see the last 50 lines of output 
tail -50 ~/Library/Logs/crm-daily.log

View the error log

Terminal — see any errors 
tail -50 ~/Library/Logs/crm-daily-error.log

Watch the log live as it runs

Terminal — follows the log in real time 
tail -f ~/Library/Logs/crm-daily.log

Common issues and fixes

Problem Fix
Script doesn't run at scheduled time Make sure Mac is awake and logged in. Try launchctl start com.changeis.crm-daily to test manually.
Error: "Permission denied" Run chmod +x '/Users/varunmalhotra/CHANGEIS Dropbox/Varun Malhotra/_Leapfrog/Corp_CRM Management/config/run-crm-daily.sh'
Error: "command not found" launchd has a limited PATH. Add export PATH=/usr/local/bin:/usr/bin:/bin:$PATH at the top of your run-crm-daily.sh script.
Log file is empty Create the Logs directory first: mkdir -p ~/Library/Logs
Need to change the time Unload, edit the plist file, then reload: launchctl unload → edit Hour in nano → launchctl load

CRM Daily Automation — Railway & Email Extension

Project: Corp CRM Daily Automation  |  Owner: Varun Malhotra  |  Date: 2026-03-26  |  Status: Draft

Overview

Extend the existing Claude Code–built CRM daily script to: (1) capture its HTML output, (2) format it with Changeis branding, (3) push it to the live Railway server, and (4) email a formatted HTML summary to Varun every morning.

Four-Phase Delivery Plan

Phase 1 — Foundation

Capture Script Output

  • Modify run-crm-daily.sh to pipe all output to a timestamped file
  • Store output in: ~/Library/Logs/crm-output-YYYY-MM-DD.txt
  • Confirm script exits with code 0 on success
  • Add error handling: if script fails, log the error clearly
Phase 2 — Branding

Format Output as HTML Report

  • Create a new script: format-crm-report.sh (or Python)
  • Wrap captured output in a full HTML page
  • Apply Changeis branding: Navy #1B2A4A, Teal #008B8B, Arial/Georgia fonts
  • Include: date header, summary stats table, full log output, Changeis footer
  • Save as: ~/Library/Logs/crm-report-YYYY-MM-DD.html
Phase 3 — Railway Push

Upload Report to Railway Server

  • Add a REST endpoint to the Railway app: POST /api/crm-report
  • Endpoint accepts: { "date": "YYYY-MM-DD", "html": "<full html string>" }
  • Railway stores reports in memory or a database (e.g. SQLite or environment variable)
  • Add a view endpoint: GET /crm-report/:date — returns the HTML report
  • The local script POSTs the formatted HTML to Railway using curl
  • Authentication: use a simple Bearer token (env variable on both sides)
Phase 4 — Email Delivery

Send HTML Report by Email

  • Use SendGrid free tier (100 emails/day free) or Gmail SMTP
  • Add email-send script: send-crm-email.sh (or Python with smtplib)
  • Email: To varun@changeis.com, Subject: "CRM Daily Report — [Date]"
  • Body: the branded HTML report (inline HTML email)
  • Credentials stored in: ~/.crm-secrets (not in Dropbox)
  • Chain: run-crm-daily.sh → format-crm-report.sh → push-to-railway.sh → send-crm-email.sh
