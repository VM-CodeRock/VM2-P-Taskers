Section 1

How To — Getting Started & Daily Use

What This Skill Does

The ccc-tee-time skill is a fully automated tee-time booking agent for Congressional Country Club (CCC). It runs two kinds of bookings: a standing Saturday autopilot that races to secure 8:44 AM Blue Course at 3:00 PM every Thursday, and on-demand bookings for any date, course, and party Varun specifies mid-week. The skill handles the entire workflow — logging in, navigating CCC's single-page booking app, filling the foursome, switching transportation to Walking, submitting, creating the Google Calendar event, and sending a push notification — with no manual steps required from Varun once the weekly Wednesday confirmation has been sent. It integrates with the local Comet browser (session-aware) and falls back to stored credentials when needed. Version 1.1 adds a self-documenting update workflow so the skill can evolve without breaking the automation.

How Varun Uses It Day-to-Day

The Saturday Autobook Routine

Caddie Note Pre-noon Blue Course bookings require a caddie. The booking flow does not arrange this automatically — Varun should contact the CCC caddie master / pro shop separately after receiving the booking confirmation.

The autobook cycle has two steps, each triggered by a scheduled cron task:

  1. Wednesday 6:00 PM ET — Confirmation Ping
    Varun receives an in-app + push notification: "CCC tee time — confirm for [Saturday date]?" with the default plan (8:44 AM Blue, Varun + Shyam + 2 TBDs, walking, 18 holes). He replies one of three ways:
    • GO — proceed as planned
    • SKIP — skip this week (no booking)
    • CHANGE [details] — modify the booking (e.g., "CHANGE time 9:30 AM, Gold Course")
    His response is written to /home/user/workspace/ccc_autobook_state.json. If no response is received by Thursday 2:00 PM, a re-prompt is sent.
  2. Thursday 2:55 PM ET — The Booking Race
    The skill pre-loads the CCC member portal in the local browser (5 minutes before the 3:00 PM ET tee-sheet release). The instant the target Saturday's tee sheet becomes visible, it navigates to Blue Course, walks the fallback ladder, clicks Reserve on the first available slot, fills the foursome, sets Walking for all players, and submits. A Google Calendar event is created on varunm24@gmail.com, and Varun receives a success or failure push notification.

On-Demand Bookings

Varun can request a one-off tee time at any point, for example: "Book Blue Course next Saturday 9 AM, just me and Shyam."

  1. The skill parses course, date, time, players, and walking/riding preference.
  2. It opens the CCC tee sheet in the local browser and confirms the slot has a green Reserve button (truly open).
  3. A confirm_action prompt is sent to Varun with exact booking details before anything is submitted.
  4. On approval, the skill executes the booking, captures a confirmation screenshot, and creates the Google Calendar event.

Skipping a Week

Reply SKIP to the Wednesday ping, or tell the skill any time before Thursday 3 PM: "Skip [date]." Skipped Saturdays are recorded in /home/user/workspace/ccc_autobook_blackouts.json so future Wednesday pings for those dates are suppressed automatically (a brief "Skipping per blackout" notification replaces them).

Reviewing / Cancelling

Ask the skill to "Show me my upcoming tee times" or "Cancel my booking for May 17." The skill opens the CCC reservations page in the local browser, reads the current bookings, and — for cancellations — will prompt for confirmation before removing the reservation. Calendar events are updated accordingly.

Common Pitfalls & the TBD Trap

The TBD Trap — Most Common Mistake A tee sheet row showing "Member Name + TBD, TBD, TBD" is not bookable. That member has reserved the slot but hasn't filled their group yet. The only signal for a truly open slot is a green Reserve button on a highlighted row. Never try to join a TBD row.
Pitfall What Happens Prevention / Fix
TBD trap Clicking a TBD row results in "No To Be Determined people allowed" error or the slot is already blocked Only act on rows with a green Reserve button
Default Riding CCC defaults all players to Riding; skill must explicitly switch each player to Walking Transportation step is mandatory after player selection
Caddie surprise Pre-noon Blue Course bookings require a caddie; booking flow doesn't handle this Booking confirmation notification always flags this; Varun arranges caddie separately
Session expiry Local Comet browser session may be logged out Credentials stored in skill; used as fallback on any login screen
10-minute hold timer Once Reserve is clicked, the hold expires if the form takes >10 minutes All steps (party size, players, walking, submit) targeted in under 5 minutes
Members-only early slots "No guests allowed for this tee time" error on slots before ~9:00 AM Gold Fall back to a later slot or book Varun solo; only CCC members in standard roster
Pre-3 PM navigation Navigating to target Saturday's tee sheet before 3:00 PM ET is rejected by CCC system Pre-flight parks on the nearest available date; poll for target Saturday to appear at 3:00:00 PM

PRD — Product Requirements Document

Problem Statement

Congressional Country Club tee times for popular Saturday morning slots on the Blue Course open at exactly 3:00 PM ET every Thursday, 10 days in advance. The best slots — particularly 8:44 AM — are claimed within seconds by members who are watching the clock. Varun cannot reliably compete in this manual "race" while working or otherwise occupied on Thursday afternoons. Additionally, arranging ad-hoc weekend rounds requires navigating CCC's complex single-page booking application, managing a specific foursome roster, handling system quirks (TBD restrictions, guest rules, transportation defaults), and then creating calendar invites and notifying playing partners — a 15–20 minute manual task each time.

The ccc-tee-time skill eliminates this friction entirely: it automates the Thursday booking race, handles on-demand bookings conversationally, and manages all downstream tasks (calendar, notifications) without requiring Varun to touch the CCC portal directly.

Goals & Non-Goals

Goals

  • Win the Thursday 3:00 PM tee-time race for the Saturday 8:44 AM Blue Course slot at least 90% of weeks
  • Complete all booking and calendar tasks without any manual steps from Varun after Wednesday confirmation
  • Surface failures immediately with enough context (tee sheet screenshot, ladder attempted) for Varun to act
  • Support on-demand bookings conversationally, with confirmation before execution
  • Maintain a durable state file so the automation survives agent restarts and session changes
  • Enable easy skill updates without breaking the running automation

Non-Goals

  • Booking tee times at clubs other than CCC (not in scope)
  • Managing caddie reservations (flagged in notifications; Varun arranges separately)
  • Automated cancellations without explicit Varun confirmation
  • Billing or payment flows (CCC charges are handled by member account automatically)
  • Joining or modifying bookings made by other CCC members

browser_task — Comet Local + Cloud Fallback

The booking race uses browser_task with use_local_browser={"local": true} (Comet). Varun's CCC session may already be authenticated in the local browser, making login a no-op. If the local browser is unavailable, the task falls back to a cloud browser instance using the stored credentials. The task prompt includes the full step-by-step booking flow from SKILL.md, the 10-minute hold timer constraint, and all credentials as fallback.

gcal Connector

Google Calendar integration via the user's connected gcal connector. Creates events on varunm24@gmail.com. Always invites varun@changeis.com. Does not use Outlook/M365. Calendar events are created only on successful booking confirmation — never on failure or partial booking.

Notifications

In-app + push notifications via the Perplexity notification system. Four notification types: Wednesday confirmation request, Thursday re-prompt (if no reply), booking success, and booking failure. All include a 1-line body summary. Failure notifications attach a screenshot of the full Blue Course tee sheet.

CCC Booking System — ClubEssential NetCaddy SPA

CCC's booking UI is a single-page application powered by ClubEssential NetCaddy at https://www.ccclub.org.

URL Constraint — No Deep Links The CCC tee booking system is a SPA with no per-date or per-course deep-link URLs. Every booking session must start from the member portal login page or the main tee times section and navigate through the UI interactively. Pre-loading the specific Saturday tee sheet via URL before 3:00 PM ET is not possible.

Key system behaviors to be aware of:

  • Tee times open at exactly 3:00 PM ET on Thursdays, 10 days in advance. Attempting to navigate to a future date before this moment returns no results.
  • TBD rows are blocked slots — not bookable. Only rows with a green Reserve button are open.
  • A 10-minute hold timer starts the instant Reserve is clicked. The form must be submitted within this window.
  • TBD player names are rejected: "No To Be Determined people allowed" — use members from the directory.
  • Some early-morning slots are members-only: "No guests allowed for this tee time." Fall back to a later slot.
  • Default transportation is Riding — must be explicitly switched to Walking for each player.
  • Caddies are required before 12:00 PM on the Blue Course; this is not handled in the booking flow.

Testing & Observability

Pipeline Timing Files

Each Thursday booking run writes a timing log at /home/user/workspace/pipeline_timing_[YYYYMMDD].txt. These logs capture step-by-step timing from Cron 2 launch through booking confirmation, enabling identification of slow steps across runs. After 3–5 live cycles, these files are the primary input for optimization decisions.

Screenshots

Booking confirmations and failure tee sheets are saved to /home/user/workspace/ with filenames ccc_booking_[YYYYMMDD].png (success) or ccc_failure_[YYYYMMDD].png (failure). Screenshots are referenced in the state file (last_result.screenshot_path) and in the calendar event description.

State File as Audit Log

ccc_autobook_state.json provides a lightweight audit trail: the last_result block records status, time secured, and notes for every cycle. For longer-term tracking, the golf-rounds-tracker deliverable (see portal) maintains a round-by-round history.

Manual Testing

Mid-week pipeline tests can be run by asking the skill to perform a dry-run navigation: open CCC tee sheet for a visible date (not the future target), confirm UI elements are intact, and report back without booking anything. This surfaces UI changes before the Thursday race.

  • Scheduled task time needs adjustment (DST, schedule change)
