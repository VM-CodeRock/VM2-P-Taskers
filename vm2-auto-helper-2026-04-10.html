### portal.html Entry Format (CRITICAL)
When adding entries to portal.html, EVERY row MUST use this exact format:
```html
<tr data-date="YYYY-MM-DD" data-type="Type Name">
<td class="col-date">YYYY-MM-DD</td>
<td class="col-title"><a href="filename.html">Title</a></td>
<td class="col-type"><span class="badge badge-{badgeclass}">Type Name</span></td>
<td class="col-link"><a class="link-icon" href="filename.html" title="Open">↗</a></td>
</tr>
```

**Required attributes:**
- `data-date="YYYY-MM-DD"` on the `<tr>` — this drives date filtering
- `data-type="Type Name"` on the `<tr>` — this drives type filtering
- Date displayed as `YYYY-MM-DD` (not "Mar 29" or other formats)
- All four `<td>` cells with proper classes: `col-date`, `col-title`, `col-type`, `col-link`
- Badge class options: `badge-research` (Research Brief), `badge-monitoring` (Compliance, Auto-Helper, Monitoring), `badge-brief` (Intelligence Brief, Daily Brief, Newsletter), `badge-research` (Deep Dive, Strategy, Tech, Architecture, Weekly Digest)
- Link href should be the LOCAL filename (not the full Railway URL)

**DO NOT:**
- Use raw `<tr><td>` without data attributes
- Use short date formats like "Mar 29" in the display
- Omit the link column
- Add entries outside the `<tbody>` (e.g., stray `<ul>` elements)
- Touch index.html (it's an immutable redirect stub)
