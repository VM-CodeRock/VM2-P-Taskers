VM2 System Guide

Your AI Operating System — Quick Reference
March 27, 2026
What VM2 Is
  • Your personal AI operating system built on Perplexity Computer
  • Handles: research, competitive intel, opportunity analysis, daily briefs, writing, scheduling, document creation
  • Everything produces a deliverable HTML page hosted on Railway, indexed in your Deliverables Portal
  • All deliverables are self-documenting — each contains the full reasoning chain so any session can pick up where the last one left off
How to Access
  • Deliverables Portal — vm2-p-taskers-production.up.railway.app  (vm2 / 97harry23!)
  • Perplexity Computer — perplexity.ai/computer  (your main sessions)
  • GitHub Repo — github.com/vm-coderock/VM2-P-Taskers  (source of truth)
Three Ways to Task VM2
Option 1

Direct Chat

  • Open Perplexity Computer and type your request
  • Deliverable gets pushed to GitHub + Railway automatically
Best for: immediate, interactive work where you want to guide the analysis
Option 2

Email Pipeline

  • Send email to vm2.viru@gmail.com with "VM2" in subject or top of body
  • Hourly cron picks it up and creates a dedicated session
  • You get a reply with the deliverable link when done
Best for: fire-and-forget tasks, delegating while you're in meetings
Option 3

Auto-Helper (Todoist)

  • Daily at 7am, VM2 scans your Todoist and shows tasks it can handle
  • Open the Auto-Helper page on Railway and click "Task VM2" on any task
  • This sends an email that triggers the pipeline above
Best for: working through your backlog systematically
How to Follow Up on Any Deliverable
1
Open any deliverable on Railway
2
Scroll to the bottom — every page has a "Continue This Work" panel
3
Choose one of three options:
  • Copy Follow-Up Prompt → paste into a new Perplexity Computer session. The new session reads the full context from the deliverable URL — all prior reasoning, findings, dead ends, and open questions.
  • Email to VM2 → sends a follow-up email that the pipeline picks up with full context
  • Create Todoist Follow-Up → adds a follow-up task to your Todoist with vm2 label
4
The new session reads the original deliverable and picks up exactly where the last session left off — no starting from scratch, no lost reasoning

Key Rules (What VM2 Always Does)
  • Every deliverable → pushed to GitHub as plain HTML (no encryption)
  • Every deliverable → logged in portal.html (the Deliverables Index)
  • Every deliverable → includes context slug for follow-up continuity
  • Railway basic auth protects everything server-side
  • portal.html = the index. index.html = redirect only. Never edit index.html.
