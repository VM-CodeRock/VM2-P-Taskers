1. System Overview

VM2 is a fully automated AI task pipeline built on Perplexity Computer. It receives work instructions via email, direct chat, or Todoist, executes them through AI agents, publishes HTML deliverables to a password-protected Railway web portal, and backs up everything to Dropbox and GitHub automatically.

Core principle: Every task produces a deliverable HTML page. Every deliverable is indexed, hosted on Railway, pushed to GitHub, and synced to Dropbox — automatically, with no manual steps required.

2.5 Todoist Integration

  • Tasks with VM2 label are eligible for pipeline execution (hourly cron)
  • Tasks with vm2-opp label are deep dive requests — created only by manual button clicks on the daily brief (see §7)
  • Daily Auto-Helper scans non-VM2 tasks and surfaces ones where AI could help
  • Completed tasks receive vm2-done label
  • API: REST v1 with pagination (https://api.todoist.com/api/v1/tasks). Connector limited to 50 results — always use direct REST API with pagination for comprehensive fetches.
  • Key labels: VM2 = 2183231010, vm2-opp = 2183369880, vm2-recurring (new v3.2)

2.5.1 Recurring Task Model (v3.2)

Recurring Todoist tasks (e.g., daily SAM checks, weekly reviews) use a three-label system:

Label ComboBehavior
VM2 onlyOne-shot task. Execute when due/overdue. Add vm2-done on completion.
VM2 + vm2-recurringExecute ONLY if due.date == today (exact match). Skip overdue — prevents catch-up loops. vm2-done label is ignored for eligibility.
VM2 + vm2-done (no recurring)Completed one-shot. Skipped permanently.

Delivery Log: After each recurring execution, the cron appends a timestamped link to the task’s description in Todoist:

── VM2 Delivery Log ──
✓ 2026-04-06 — https://vm2-p-taskers-production.up.railway.app/faa-daily-sam-2026-04-06.html
✓ 2026-04-07 — https://vm2-p-taskers-production.up.railway.app/faa-daily-sam-2026-04-07.html

This builds a running history of all deliverables directly on the task — no portal searching needed.

State tracking: /home/user/workspace/cron_tracking/todoist_vm2/recurring_state.json records {task_id: last_execution_date} to prevent double-runs within the same day (hourly cron).

4.2 Sync Mechanism

The nightly vm2-wrapup skill (v2.1+) performs a full diff sync:

  1. Lists all .html files in /VM2-main-folder/VM2-P/Taskers/ recursively
  2. Lists all deliverable HTML files in the GitHub repo
  3. Computes the diff (GitHub − Dropbox)
  4. Uploads all missing files to the correct categorized subfolder
  5. Reports N synced, N already present in the compliance report

5. GitHub & Railway Hosting

5.1 Push Workflow

# Standard deliverable push
git add <filename>.html portal.html
git commit -m "Deliverable: <slug> · VM2 · YYYY-MM-DD"
git pull --rebase origin main   # resolve any concurrent pushes
git push origin main
# Railway auto-deploys within ~60 seconds
Conflict resolution: Multiple crons may push concurrently. Always use git pull --rebase before pushing. If rebase fails, the cron escalates to the foreground session for manual resolution.

5.2 HTML Requirements

  • All HTML must be plain HTML — no StaticCrypt encryption
  • Railway nginx basic auth protects all pages server-side
  • Deliverables should include a "Continue This Work" footer panel with context for follow-up sessions
