VM2 Knowledge Workflow Brief
Using Every’s “Compound Knowledge” plugin as a second brain + VM2 runbooks
Date: 2026-03-25
Outcome: Practical setup + how to apply to govcon + Changeis
Primary reference: EveryInc/compound-knowledge-plugin

1) What it is (in one paragraph)

Every’s Compound Knowledge plugin is a set of Claude Code workflows for knowledge work that deliberately loops through brainstorm → plan → review → execute → save learnings, so the system improves over time.

Core loop (from the repo):

/kw:brainstorm → /kw:plan → /kw:confidence (optional) → /kw:review → /kw:work → /kw:compound

Source: workflow command list in EveryInc/compound-knowledge-plugin.

2) Minimal setup (30 minutes)

The plugin is designed to work with any project that has a plans/ directory and stores learnings in docs/knowledge/ as plain markdown with YAML frontmatter (repo).

ItemWhat you put there (recommended for VM2)
CLAUDE.md Changeis context, govcon priorities, qualification rules, preferred brief/email formats, and a data-source hierarchy (the repo says workflows read this file for business context, data sources, and conventions).
plans/ One plan per problem/opportunity (e.g., “FAA EITSS-2 tracking plan”).
docs/knowledge/ Second-brain “nuggets”: runbooks, templates, insights, decision logs (this is where /kw:compound writes learnings per the repo).

Source: storage / configuration notes in EveryInc/compound-knowledge-plugin.

3) How to apply it to your recurring VM2 work

Think of each recurring VM2 task as a runbook that compounds. Each time you run it, you update the runbook with better filters, better sources, and fewer false positives.

  • FAA/DOT opportunity monitoring: compound search filters that reliably surface FAA IT / acquisition activity, plus “what to ignore” patterns.
  • Events tracking (ACT-IAC, PSC, etc.): compound a vetted list of event pages + “what’s worth attending” heuristics for you (role-based, topic-based).
  • Internal enablement (AI Champions, training programs): compound templates for internal comms + a “Changeis AI literacy ladder” so each campaign gets easier.

Why it works: the repo describes /kw:plan as searching docs/knowledge/ for past learnings saved by /kw:compound, and /kw:compound as saving 1–3 learnings with YAML frontmatter and checking for contradictions with stale knowledge (repo).

4) “CLAUDE.md” sections I’d include for Changeis

  • Mission & positioning: what Changeis does, where you win.
  • Priority buyer map: FAA (offices you care about), DOT modes, adjacent agencies.
  • Opportunity qualification rubric: vehicle, timing, ceiling, set-aside, incumbent risk, subcontracting path.
  • Data-source hierarchy: what counts as authoritative (the plugin’s data accuracy reviewer cares about sourcing per the repo).
  • Deliverable templates: 1-page exec brief format; email format; call-prep format.

5) Install + first run (Claude Code)

Install steps shown in the repo:

/plugin marketplace add EveryInc/compound-knowledge-plugin /plugin install compound-knowledge

Starter loop from the repo:

/kw:brainstorm I need to figure out our Q2 content strategy /kw:plan /kw:compound
