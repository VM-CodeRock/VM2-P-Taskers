1) What Bauplan is (plain-English)

Bauplan positions itself as a Python-first, serverless âlakehouse runtimeâ for building and operating data pipelines directly on object storage (e.g., S3), using Git-style branching and versioning for data so changes can be run and validated safely before being published.

Core workflow: create a data branch â run your transforms â validate outputs â publish to main via an atomic merge with rollback to a known-good commit.

How they describe the foundations

  • Open lakehouse foundation: Apache Iceberg on object storage for transactional guarantees, schema evolution, and time travel.
  • Serverless compute: function-as-a-service execution model; you write pure Python while the platform handles isolation and scaling.
  • Interoperable outputs: curated tables remain portable Iceberg tables in your storage, accessible to other engines/tools.
