1 What Is OpenClaw?

🦞 The short version for Varun

OpenClaw is the fastest-growing open-source AI project in history — 335,000 GitHub stars in 60 days, surpassing React's 10-year record. It is a personal AI assistant that runs 24/7 on your own machine. It connects to WhatsApp, Telegram, iMessage, Slack, Discord, and 15+ other channels. It runs your "Skills" (like the ones you already use in Perplexity), fires cron jobs automatically, and maintains memory across all sessions. Critically: you can swap in any AI model including free local ones.

OpenClaw was created by Peter Steinberger (creator of PSPDFKit) in November 2025 as a weekend project. It rebranded from "Clawdbot" in January 2026 and exploded after Jensen Huang (Nvidia CEO) called it "the next ChatGPT."

The 4 Core Powers of OpenClaw

Feature What it does Equivalent in your current setup
Memory Remembers everything about you — preferences, past conversations, recurring projects. Stored in local Markdown files in ~/.openclaw/workspace/ Claude's context window (but persistent across ALL sessions)
Heartbeat AI wakes up proactively every 30 min. Checks your inbox, runs pending tasks, sends you summaries — without you asking Perplexity cron jobs, but smarter and automatic
Skills 1,100+ community plugins from ClawHub marketplace. SKILL.md files in your workspace = same format as Perplexity skills. Fully portable. Your existing Perplexity Computer skills
Cron Jobs Schedule any task (daily briefings, email summaries, research runs) using natural language or cron syntax Perplexity Computer routines
✅ Key advantage for you

OpenClaw's Skills are stored as plain SKILL.md files — almost identical to the format used by Perplexity Compute. Your existing skills and routines migrate directly. You just point OpenClaw at your skills folder.

⚠ Security note

OpenClaw is powerful but needs careful setup. In early 2026, 135,000 exposed instances were found with a remote code execution vulnerability (CVE-2026-25253). Run it behind Tailscale or SSH tunnel — never expose the gateway port directly to the internet. This guide covers safe setup.

3 Top Picks for Your Setup

🥇 #1 Recommendation: Qwen2.5-Coder-32B Best Coder Runs on Mac Mini 64GB
The #1 open-source coding model as of March 2026. Beats GPT-4o on HumanEval (92.7%), LiveCodeBench, and SQL benchmarks. Runs in Ollama on a 64GB Mac Mini at ~10 tokens/sec (usable for coding). It is the open-source Claude Code equivalent for pure coding tasks. Apache 2.0 licensed — free forever.

Caveat: Made by Alibaba (Chinese company), but weights are fully open and run 100% locally. Your data never leaves your machine.
🥈 #2 Recommendation: Llama 4 Scout US-Made Best for Mac Mini 32–64GB
Meta's Scout model is a Mixture-of-Experts (MoE) model — 109B total parameters but only 17B active at a time. This means it runs like a 17B model but thinks like a much larger one. The 10M context window is a game-changer for long coding sessions. US-made, Llama license allows commercial use. Best all-rounder for OpenClaw's multi-task environment.
🥉 #3 Recommendation: Phi-4 (14B) US-Made (Microsoft) Fastest Local
The lightest option that still punches above its weight. Runs at 25–30 tokens/sec on Mac Mini M4, making it ideal for always-on OpenClaw heartbeat checks, cron jobs, and routing tasks. MIT licensed. Best for your Hostinger VPS (CPU-only inference) and as the "quick check" agent in a multi-agent chain.

4 Multi-Agent Frameworks

OpenClaw has built-in multi-agent support via Sessions routing — you can already run multiple isolated agents from one OpenClaw gateway. For more complex orchestration, here are your options:

Framework Monthly Searches Ease of Use Ollama Support Best For Verdict
OpenClaw Sessions — Easiest ✅ Native Multiple personality agents (coder, researcher, assistant) Start here — already built in
CrewAI 14,800 Beginner-friendly ✅ Yes Task delegation, parallel workflows, report generation Best option if you want to add Python-based agents
LangGraph 27,100 Intermediate ✅ Yes Complex multi-step reasoning flows with state management Most powerful but needs coding experience
AutoGen (Microsoft) — Intermediate ✅ Yes Conversational agent teams — agents talk to each other Good for automated research pipelines
✅ Varun's Multi-Agent Plan (non-technical path)

You don't need to set up CrewAI or LangGraph initially. OpenClaw's built-in Sessions routing handles 90% of multi-agent needs. You create named sessions (Coder-Claw, Research-Claw, Quick-Claw) each with different model assignments in your config file. Each session maintains its own memory, context, and tools.

OpenClaw Multi-Agent Config (the easy way)

~/.openclaw/config.yaml — Multiple Agent Sessions
# Each "session" is an independent agent with its own model and tools agents: main-claw: model: "ollama/llama4-scout" # Your primary heavy-duty agent context: 128000 # Large context for complex tasks tools: [browser, cron, files, email] quick-claw: model: "ollama/phi4" # Fast replies for WhatsApp/Telegram context: 8192 tools: [email_send, calendar] coder-claw: model: "ollama/qwen2.5-coder:32b" # Dedicated coding agent context: 32768 tools: [browser, files, bash, github]

5 Mac Mini Setup — Step by Step

What you need

A Mac Mini with Apple Silicon (M1, M2, M3, or M4). Minimum 24 GB RAM for most models, 64 GB strongly recommended for Qwen 32B at full quality. The Mac Mini M4 Pro with 64 GB is ideal for this setup.

Step 1 — Install Homebrew (Mac package manager)

Homebrew lets you install software with simple commands. Open the Terminal app (search "Terminal" in Spotlight) and paste this:

Terminal — Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Follow the prompts. It will ask for your Mac password. # This takes 3-5 minutes. You only do this once.

Step 2 — Install Ollama (your local AI engine)

Ollama is the software that downloads and runs AI models on your Mac. Think of it as iTunes for AI models.

Terminal — Install Ollama
brew install --cask ollama # This installs Ollama as a menu bar app that runs in the background # You'll see a llama icon in your menu bar when it's running

Step 3 — Download Your AI Models

Now download the models you want to use. Each command downloads the model (this takes 10–30 minutes depending on your internet). Pick based on your RAM:

Your Mac Mini RAMBest Model to DownloadDownload Size
24 GBPhi-4 (14B) or Mistral Small 3~9–15 GB
32 GBLlama 4 Scout (recommended)~20 GB
64 GBQwen2.5-Coder-32B + Llama 4 Scout~40 GB total
Terminal — Download Models (run the ones matching your RAM)
# FOR 24 GB MAC MINI: Download Phi-4 (fast, smart, fits perfectly) ollama pull phi4 # FOR 32-64 GB MAC MINI: Download Llama 4 Scout (US-made, excellent) ollama pull llama4-scout # FOR 64 GB MAC MINI: Also download the best coding model ollama pull qwen2.5-coder:32b # Test that it works (type a message, press Enter, type /bye to exit) ollama run llama4-scout

Step 4 — Install Node.js (needed for OpenClaw)

Terminal — Install Node.js
brew install node@24 # Verify it worked: node --version # Should print: v24.x.x

Step 5 — Install OpenClaw

Terminal — Install and Configure OpenClaw
# Install OpenClaw globally npm install -g @openclaw/openclaw # Run the setup wizard — it will detect Ollama automatically openclaw onboard # When asked for a model provider, choose: Ollama (Local) # When asked for base URL, press Enter (default: http://127.0.0.1:11434) # Choose "Local only" mode (free, private, no API keys needed) # OR: One-line non-interactive setup with Llama 4 Scout openclaw onboard --non-interactive \ --auth-choice ollama \ --custom-base-url "http://127.0.0.1:11434" \ --custom-model-id "llama4-scout" \ --accept-risk

Step 6 — Start OpenClaw

Terminal — Start OpenClaw Gateway
# Start the gateway in the background openclaw gateway start # To make it start automatically on boot: openclaw gateway start --daemon # Check status anytime: openclaw doctor

Step 7 — Connect OpenClaw to Telegram or WhatsApp

OpenClaw connects to your messaging apps so you can chat with your AI from your phone:

Terminal — Add Telegram Channel
# First, create a bot via @BotFather on Telegram and get your token # Then add it to OpenClaw: openclaw channel add telegram --token YOUR_BOT_TOKEN_HERE # Now send a message to your bot on Telegram — OpenClaw will reply!

6 Hostinger VPS Setup

Role of Hostinger in your setup

Your Hostinger VPS is the always-on server — it keeps OpenClaw running even when your Mac Mini is asleep or off. Think of it as the "spine" that routes messages, runs heartbeat/cron jobs, and handles lightweight tasks 24/7. It does NOT need a GPU — it uses small CPU-friendly models or forwards heavy requests to your Mac Mini.

Recommended Hostinger Plan

Plan Specs Promo Price Renewal Price Use For
KVM 2 (recommended) 2 vCPU, 8 GB RAM, 100 GB NVMe $6.99/mo $12.99/mo OpenClaw gateway + Phi-4 lightweight model (CPU)
KVM 4 (if you want faster CPU inference) 4 vCPU, 16 GB RAM, 200 GB NVMe $9.99/mo $24.99/mo Slightly faster, run 7B models locally on CPU

Step-by-Step Hostinger Setup

Create Your VPS on Hostinger
Go to hostinger.com/vps-hosting, choose KVM 2, select Ubuntu 24.04 as the OS. Write down the IP address and root password Hostinger gives you.
Connect to Your Server
On your Mac, open Terminal and type:
ssh root@YOUR_SERVER_IP_ADDRESS
Enter your password when asked. You're now inside the server.
Install Required Software
Run these commands one by one on the server:
Hostinger Server — Setup Commands
# Update the server first apt update && apt upgrade -y # Install Node.js 24 curl -fsSL https://deb.nodesource.com/setup_24.x | bash - apt install -y nodejs # Install Ollama (for CPU-only small models) curl -fsSL https://ollama.com/install.sh | sh # Download a small model that runs on CPU (Phi-4 is ideal for this) ollama pull phi4 # Install OpenClaw npm install -g @openclaw/openclaw # Configure OpenClaw to use local Phi-4 AND forward heavy tasks to Mac Mini openclaw onboard --non-interactive \ --auth-choice ollama \ --custom-base-url "http://127.0.0.1:11434" \ --custom-model-id "phi4" \ --accept-risk
Secure with Tailscale (important!)
Install Tailscale on both your Mac Mini AND the Hostinger server so they can talk to each other securely. This also lets you access the server safely from anywhere.
Both Machines — Install Tailscale
# Run on BOTH Mac Mini AND Hostinger server curl -fsSL https://tailscale.com/install.sh | sh tailscale up # Follow the link it shows — sign in with your Google/GitHub account # Both machines will get Tailscale IPs like 100.x.x.x # Now they can talk to each other privately, even behind NAT
Configure Hostinger to Route Heavy Tasks to Mac Mini
In your OpenClaw config, set the fallback model to point to your Mac Mini's Ollama via Tailscale:
Hostinger ~/.openclaw/config.yaml — Fallback to Mac Mini
models: providers: ollama-local: baseUrl: "http://127.0.0.1:11434" # Local Phi-4 on VPS apiKey: "ollama-local" ollama-mac: baseUrl: "http://100.x.x.x:11434" # Your Mac Mini Tailscale IP apiKey: "ollama-local" agents: defaults: model: primary: "ollama-local/phi4" # Fast, cheap tasks fallbacks: ["ollama-mac/llama4-scout"] # Heavy tasks → Mac

9 Migration Roadmap

How to move your existing skills, routines, and cron jobs from Perplexity Compute / Claude Code to your new self-hosted setup.

✅ The good news: Your skills ARE portable

OpenClaw uses SKILL.md files in its workspace folder. Perplexity Compute skills are the same SKILL.md format. Direct copy-paste migration is supported. You keep all your routines — they just run locally for free.

Phase 1 — Migrate Skills (Week 1)

Export your Perplexity skills
In Perplexity Compute, find your skills folder. All skills are stored as SKILL.md files. Copy this entire folder to your Downloads.
Copy skills to OpenClaw workspace
Terminal — Copy skills to OpenClaw
# Create the OpenClaw skills directory mkdir -p ~/.openclaw/workspace/skills/ # Copy your existing skills (replace ~/Downloads/my-skills with your path) cp -r ~/Downloads/my-skills/* ~/.openclaw/workspace/skills/ # Verify skills are loaded: openclaw skills list
Test each skill
Send a message to your OpenClaw agent asking it to use each migrated skill. They should work immediately since the format is compatible.

Phase 2 — Migrate Cron Jobs (Week 1–2)

Your cron jobs from Perplexity Compute use the same scheduling format as OpenClaw. Copy them to your OpenClaw cron config:

~/.openclaw/workspace/crons.yaml — Your Scheduled Jobs
# Same cron syntax you already know crons: - name: "daily-briefing" schedule: "0 8 * * *" # 8am every day prompt: "Prepare my morning briefing: check email, calendar, top news" model: "ollama/llama4-scout" - name: "weekly-vm2-sync" schedule: "0 9 * * MON" # 9am every Monday prompt: "Run VM2 weekly review: check GitHub tasks, summarize progress" model: "ollama/phi4" # Fast model for routine tasks

Phase 3 — Migrate Claude Code Workflows (Week 2–3)

To replace Claude Code with a local coding agent, you have two options:

Option A: Use Cline in VS Code Recommended — Easiest
Install the Cline extension in VS Code. Connect it to Ollama with Qwen2.5-Coder-32B. Cline has the same Plan → Review → Execute workflow as Claude Code but is fully free and local.
VS Code — Install and Configure Cline with Ollama
# In VS Code, go to Extensions (⌘+Shift+X) and search "Cline" # Install the Cline extension by Cline Bot Inc # Then open Cline settings and add this configuration: { "cline.apiProvider": "ollama", "cline.ollamaBaseUrl": "http://localhost:11434", "cline.model": "qwen2.5-coder:32b", "cline.contextWindow": 32768 } # Done. Cline is now your free local Claude Code replacement.
Option B: Use OpenClaw with Claude Code (lower cost) Hybrid Approach
Keep Claude Code but route it through Ollama for 90% of tasks. For the remaining 10% of complex tasks, it falls back to the actual Claude API. This cuts your Anthropic bill by ~90% while keeping Claude's quality for hard problems.
Terminal — Run Claude Code with Local Model via Ollama
# Ollama provides an Anthropic-compatible API # This lets Claude Code talk to local models without a subscription export ANTHROPIC_AUTH_TOKEN=ollama export ANTHROPIC_BASE_URL=http://localhost:11434 # Run Claude Code with your local Llama 4 Scout model claude --model llama4-scout # Or use the Ollama launch shortcut: ollama launch claude --model qwen2.5-coder:32b
