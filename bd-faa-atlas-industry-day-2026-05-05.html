Executive Summary

Bottom Line Up Front

ATLAS is a once-in-a-decade FAA IT modernization opportunity. The April 30, 2026 Industry Day confirmed that Phase 2 is imminent, requires an OTA signature and physical data media pickup before instructions are released, and will evaluate portfolio rationalization strategy — not technical architecture. AVS Portfolio — Phase 2 Challenge Systems

The Aviation Safety (AVS) portfolio is the primary Phase 2 evaluation focus. Six interconnected systems manage the full airmen certification lifecycle — from application to certification to medical clearance. The core modernization challenge is that these systems were built independently, share no identity layer, and collectively form a brittle ecosystem serving hundreds of thousands of pilots worldwide.

Strategic Framing for Phase 2

FAA's stated priorities — Frictionless Security, Improved User Experience, Increased Efficiency — are fundamentally a service design problem layered over a technical modernization problem. The winning contractor will lead with user journey mapping and identity architecture, not infrastructure migration. This is Changeis's strongest angle.

IACRA
Integrated Airman Certification and Rating Application
Public-facing web app. 15+ certificate types, 100+ certification paths. Stack: Windows Server / SQL Server. Protected by Akamai. Paper fallback during outages creates dual records requiring manual reconciliation — a major operational pain point.
Public-Facing Legacy Stack Paper Fallback Risk
RMS
Registry Management System
IBM z/OS mainframe (CAIS/AADOCS + IMS). Official system of record for ALL airmen certificates. 146M+ document images. Natural/ADABAS programming — proprietary, end-of-life ecosystem. ACIS replacement in progress (Angular/Java/Postgres on FAA GovCloud), targeting production summer 2026. Highest technical risk in the portfolio.
z/OS Mainframe ADABAS/Natural 146M+ Images ACIS Transition Active
MedXPress
FAA Medical Certificate Portal
Public-facing medical certification portal. Form 8500-8 submission. Email/password login — no SSO, no Login.gov. Hundreds of thousands of worldwide users. Future goals: Login.gov SSO, mobile responsiveness, real-time notifications, tighter IACRA/DIWS integration. Identity modernization is the first priority.
Public-Facing No SSO Login.gov Target Mobile Gap
AMCS
Aerospace Medical Certification Subsystem
AME (Aviation Medical Examiner) portal. Login.gov integration already implemented. Thousands of worldwide users — international AMEs, HIMS AMEs, flight surgeons. More modern posture than MedXPress; serves as the model for the broader identity consolidation strategy.
AME-Facing Login.gov ✓ Model for SSO
DIWS
Document Imaging Workflow System
Internal system of record for medical certificates. ePower workflow engine. PIV/MyAccess authentication. CPDSS (Covered Position Decision Support System) for ATC medical clearances is being integrated into DIWS — consolidating one more siloed system. Workflow engine modernization is the key technical challenge.
Internal PIV/MyAccess ePower Engine CPDSS Integration
DMS
Designee Management System
Manages 10,000+ active FAA designees (DPEs, DMEs, etc.). Identity management complexity is high — designees span civil/military, domestic/international, multiple certificate types. Excessive Letters of Authorization (LOAs) and manual administrative overhead are the primary user complaints.
10K+ Active Designees LOA Overhead Identity Complexity

5 Most Important Industry Day Takeaways

01
Phase 2 Gated by OTA + Physical Media — No Exceptions
FAA will not release Phase 2 instructions, data, or evaluation criteria until the OTA is signed AND physical media is picked up in person at FAA HQ. This is a hard process gate — not procedural. Every day of delay on OTA execution is a day lost on Phase 2 prep time.
02
Service Design Problem, Not Just a Tech Refresh
FAA explicitly wants "intuitive, trustworthy, low-friction digital services" for a complex global user base. The winning team leads with user journey mapping and change management — not infrastructure. This is Changeis's strongest differentiator vs. traditional IT shops.
03
Migration Execution > Target Architecture
FAA repeatedly emphasized practical, proven transition planning over aspirational end-state blueprints. Phase 2 evaluations will reward detailed migration sequencing, risk mitigation plans, and realistic timelines — not theoretical modernization frameworks.
04
Identity / SSO Is a Major Discriminator — But Hard
Minors, foreign users, legacy authentication systems, and mission-critical uptime make unified identity far more complex than a Login.gov API call. The contractor that presents a credible, phased identity consolidation plan (with AMCS as the working proof point) will score highest on this dimension.
05
Continuous Operating Model — Not a One-Time Build
Constant regulatory and policy changes require a delivery model that absorbs rapid change under FFP pricing. Sustainment + enhancement agility is as important as initial modernization. Changeis's change management model and experience with regulatory-driven update cycles is a direct answer to this requirement.
