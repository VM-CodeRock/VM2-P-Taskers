2 · What Is Conway

Conway is Anthropic's internally-tested, always-on agentic AI platform — a fundamental architectural departure from Claude's existing prompt-response model. Rather than waiting for a user to type a query, Conway operates as a persistent background process that monitors subscribed event streams, wakes on webhook triggers, and autonomously executes multi-step workflows.

The system was first disclosed on April 1, 2026, through a testing catalog leak reported by TestingCatalog and confirmed by multiple outlets. Anthropic has not officially announced Conway, and it remains in internal testing. The name is believed to reference Conway's Game of Life — apt for a system that evolves and acts from simple rules without constant human direction.

Conway vs. Standard Claude

From Reactive to Proactive

  • Standard Claude: Waits for user prompt; single-session context; conversation ends when user leaves
  • Conway: Runs 24/7; persists across sessions; self-initiates on triggers; completes work while user is offline
  • Closer to a daemon process than a chatbot — always listening, always ready
Product Ecosystem Position

Anthropic's Three-Agent Stack

  • Claude Code: Terminal-based AI for software engineers; coding, testing, refactoring
  • Cowork: Desktop agent for non-technical enterprise workers; general productivity workflows
  • Conway: Always-on power layer for autonomous multi-step execution, custom extensions, event-driven automation

3 · Technical Architecture

Based on leaked testing catalog reports and code analysis, Conway is built around five integrated layers that together create a persistent agentic runtime — not merely a wrapper on top of an LLM API.

Reasoning Core — Claude 4.6 / Sonnet
LLM inference backbone; 200K–1M token context; optimized for agentic long-horizon reasoning and multi-step planning; Anthropic's Constitutional AI safety layer

Full Chrome automation via dedicated integration toggle; navigates, clicks, fills forms, extracts data autonomously; web task execution without user presence

CNW Extension Framework
Proprietary ZIP-based extension packaging; developers build custom tools, UI panels, context handlers; analogous to Salesforce AppExchange; integrates with Anthropic MCP (Model Context Protocol)
↑ ↓
💾
Persistent Memory Store
Cross-session context retention; tracks actions taken, decisions made, open tasks; maintains awareness across days or weeks; feeds back into reasoning core for continuity

User Interface: Conway features a dedicated three-panel sidebar UI — Search, Chat, and System — distinct from Claude.ai's chat interface. This is not a chat window; it is a task management and monitoring console where work is assigned, tracked, and reviewed. Claude Code is embedded directly in the UI for seamless code execution alongside natural language tasks.

MCP Integration Conway's CNW extension framework aligns directly with Anthropic's Model Context Protocol (MCP) — the open standard for connecting AI models to external data sources and tools. This means any MCP-compatible integration (databases, APIs, internal systems) can be wired into a Conway workflow without custom CNW extension development, a significant advantage for federal IT environments with existing MCP-compatible tool catalogs.

FedRAMP Authorization Details

  • ATO Holder: Palantir Federal Cloud Service – Supporting Services (PFCS-SS) holds the C4G FedRAMP High ATO
  • Independent Assessor: Schellman Compliance, LLC performed the 3PAO assessment
  • Data Residency: Logically isolated from commercial tenants; hosted through government-authorized partner infrastructure; supports agency boundary requirements
  • Encryption: TLS in transit; AES-256 at rest

  • ITAR: Only via Bedrock IL5 — not C4G directly

FISMA & Data Residency Considerations

Claude for Government's FedRAMP High ATO satisfies FISMA High system requirements for unclassified sensitive data. FISMA compliance is agency-managed — Anthropic's ATO documentation is available via the FedRAMP Package Access Request Form and through Anthropic's Trust Center.
