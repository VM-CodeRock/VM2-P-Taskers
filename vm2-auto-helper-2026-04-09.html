Original Instructions

Daily Todoist Auto-Helper: Scan ALL Todoist tasks (not just VM2-labeled ones) and identify tasks where AI could help. This is NOT for executing VM2 tasks — that's a separate hourly cron.

PROCEDURE:
1. Load skills: changeis-style, research-assistant
2. Read deliverable-specs.md for context slug spec
3. Use Todoist connector (todoist-search-tasks) to fetch ALL tasks across all projects
4. Filter OUT tasks that already have the VM2 label (label ID: 2183231010) or vm2-done label
5. Analyze remaining tasks for where AI can help
6. Generate vm2-auto-helper-YYYY-MM-DD.html with Changeis styling + mailto buttons + context slug panel
7. Push HTML to GitHub repo vm-coderock/VM2-P-Taskers
8. Update portal.html with new entry
9. Save to Dropbox at V M2/VM2-main-folder/VM2-P/

Approach

Fetched tasks and projects from Todoist, filtered out tasks with VM2-related labels, then used simple heuristics (keywords + task patterns) to identify tasks likely to benefit from AI assistance and generated mailto buttons to delegate.

Key Findings

  • Fetched 50 active tasks total.
  • Filtered to 45 tasks after excluding VM2-labeled tasks.
  • Flagged 35 tasks as good candidates for AI help.

Dead Ends

  • Org skill 'changeis-style' was not available, so styling was implemented to match deliverable-specs.md (navy topbar, dark/light toggle, Charter/Georgia headings).

Assumptions

  • Tasks that are clearly birthdays/anniversaries or generic habit goals were excluded as low-leverage for AI delegation.
  • Todoist connector appears to return up to 50 tasks per call; this run used one call (may not include tasks beyond the first page if more exist).

Open Questions

  • If there are more than 50 active tasks, add paging/multiple queries to ensure full coverage.

Data Sources

  • Todoist API Tasks endpoint

Next Steps

  • Implement pagination / per-project iteration if needed to ensure all tasks are included.
  • Tune heuristics once we see false positives/negatives.
