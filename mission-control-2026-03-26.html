1) What вАЬMission ControlвАЭ should do (use cases)

  • Single pane of glass: active sessions, queued work, completed work, failures.
  • Work orchestration: queue вЖТ route to the right agent вЖТ track WIP limits, deadlines, dependencies.
  • Governance & safety: approval gates for irreversible actions (send email, post, purchase, delete).
  • Evidence + audit trail: store sources, decisions, prompts, tool calls, and artifacts per deliverable.
  • Learning loop: capture what worked, convert into templates + reusable workflows.
  • Outcome view: map agent output to business goals (pipeline, capture, hiring velocity, compliance).

Suggested information architecture

Dashboard
  • Now: running jobs/sessions
  • Next: due tasks + priorities
  • Recent: deliverables library
Ops / Health
  • cron list + last success + errors
  • connector health (Todoist/GitHub/etc.)
  • spend/usage indicators
Tasks & Queue
  • Kanban: Backlog / In progress / Needs approval / Done
  • templates for repeatable tasks
Guardrails
  • approval queue with full draft content
  • role-based permissions per agent

Implementation plan (phased)

  1. Phase 1 (MVP): lightweight UI + JSON state + activity log.
  2. Phase 2: integrate Todoist + GitHub Pages index + (if available) session exports.
  3. Phase 3: approvals + audit log + permission profiles.
  4. Phase 4: analytics (throughput, cycle time, failure rate) + impact mapping.
