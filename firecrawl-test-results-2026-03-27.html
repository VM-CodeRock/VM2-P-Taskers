SAM.gov API Reality Check
All three SAM.gov API endpoints tested returned HTTP 404 without a registered API key. The SAM.gov API requires:
  • A registered sam.gov account with a verified UEI
  • API key request through the profile system (1–5 business day wait)
  • Mandatory date range parameters on every search call
  • Understanding of undocumented pagination quirks
  • Rate limit: 10 requests/second, 1,000/day on the free tier

Key Finding: DHS.gov is Server-Side Rendered (Drupal)
DHS.gov runs Drupal — a server-side CMS, not a JS SPA. This means static HTTP delivers more raw bytes than Firecrawl (267K vs 194K) because Firecrawl's markdown conversion strips HTML boilerplate. For DHS.gov content pages, static HTTP is faster and sufficient. Firecrawl's value would be highest on APFS-style portals built on React/Angular that require browser execution.
