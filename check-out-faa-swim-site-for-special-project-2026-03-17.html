FAA SWIM — Special Project Brief

Prepared for Varun Malhotra · 2026-03-17
Quick take
SWIM is the FAA’s data-sharing backbone for NextGen: data producers publish once, consumers connect once, and authorized users receive near real-time aviation information (flight, aeronautical, weather, surveillance).

1) What SWIM is

  • FAA System Wide Information Management (SWIM) is a National Airspace System (NAS)-wide information system supporting NextGen and acting as the digital data-sharing backbone, offering a single point of access where data producers publish once and consumers connect once. (FAA SWIM Program Overview)

2) What SWIM provides (information domains)

  • Designed to provide access to near real-time aeronautical, flight, and weather information to stakeholders inside and outside the NAS. (FAA SWIM Program Overview)

3) Key data products / services (examples)

SFDPS (en route flight data)
  • Provides en route flight data to NAS consumers and supports real-time analytics, business processes, and research. (FAA SFDPS)
  • Pipeline: ERAM → HADDS/EDDS → SFDPS transforms Common Message Set into SWIM-compliant XML and publishes via NEMS; external access is via NESG. (FAA SFDPS)
  • Service types include En Route Flight / Airspace / General Message / Operational data publication & query. (FAA SFDPS)
STDDS (terminal / surface & tower/TRACON feeds)
  • Converts legacy terminal data from towers/TRACONs into accessible information published via NEMS. (FAA STDDS)
  • Publishes data from ASDE-X, ASSC, STARS, RVR, EFSTS, and TDLS. (FAA STDDS)
  • Installed at 38 TRACONs; provides access to data from 200+ airports and 400+ individual systems. (FAA STDDS)

4) How to get access (SWIFT Portal / SCDS)

  • The SWIFT Portal is described by FAA as a publicly accessible cloud-based system providing near real-time SWIM data via Solace JMS messaging. (FAA Getting Access to SWIM)
  • Some services are not supported in SWIFT Portal; FAA directs requests for CDM Data Elements and certain TFMS capabilities to Data-To-Industry@faa.gov. (FAA Getting Access to SWIM)
  • FAA points users to NSRR for service documentation and to the support site for FAQs (support.swim.faa.gov). (FAA Getting Access to SWIM)

5) Practical use cases (govcon / tech consulting)

Operational dashboards & situational awareness
Fuse en route + terminal feeds to highlight constraints, surface movement hotspots, and capacity impacts.
Data engineering & interoperability
Build “connect once” integrations and services around SWIM patterns (publish/subscribe + query) for enterprise reuse.
Research, performance analysis, evaluation
Use SFDPS publication + query patterns to support studies (delay attribution, trajectory-based ops, post-event analysis).
Notes: These use cases are framed from SWIM’s stated purpose and the example products above. (FAA SWIM Program Overview, FAA SFDPS, FAA STDDS)

6) Recommended next step (for a “special project”)

  1. Identify which SWIM service(s) and message schemas you need (often via NSRR / SWIFT Portal documentation). (FAA Getting Access to SWIM)
  2. Confirm onboarding path: SWIFT Portal (cloud/JMS) vs NESG vs Data-To-Industry route for unsupported services. (FAA Getting Access to SWIM)
