1. Executive Summary

⚠ Headline Answer: NO — Not for SAM.gov Attachments

Bottom line: Browser Use and comparable AI browser automation tools are explicitly prohibited by SAM.gov's Terms of Use for automated data gathering, web scraping, and bot-driven downloads. Using these tools — including against the login-walled attachment/document area — would violate SAM.gov ToS, risk immediate account termination via Login.gov, and expose Changeis to liability under the Computer Fraud and Abuse Act (CFAA). As a federal contractor, the compliance bar is exceptionally high. The correct path is the official SAM.gov Opportunities API plus supplemental intelligence platforms (HigherGov, GovWin).

However, Browser Use and competitors are genuinely valuable for other Changeis workflows where no ToS prohibition exists: partner portal automation, HigherGov/GovWin intelligence gathering (subject to their ToS), Costpoint internal workflows, LinkedIn research, and competitive intelligence on non-federal commercial sites. The technology is real, capable, and cost-effective — it is simply the wrong tool for the specific SAM.gov attachment download problem.

2. Browser Use — Product Profile

What Is It?

Browser Use is an open-source Python library (MIT license) created by Magnus Müller and Gregor Žunić that enables AI agents to control a real web browser in natural language. It became one of the fastest-growing AI repositories on GitHub in late 2024. The project pairs Playwright (Microsoft's browser automation engine) with any LLM you choose — GPT-4o, Claude Sonnet/Opus, Gemini, or local models via Ollama — creating a perception-action loop that can read pages, click, type, scroll, fill forms, and download files.

3. SAM.gov Restrictions Today

The Login Wall Problem

Contracting officers on SAM.gov post solicitation attachments — Statements of Work, Sections L/M, PWS documents, PPQ templates — that constitute the most valuable proposal intelligence. As of 2024–2025, SAM.gov requires a Login.gov-authenticated account to view and download these attachments in the browser UI, even when the parent opportunity is publicly visible. This creates a perceived "wall" that drives contractors to look for automation workarounds.

Important distinction: The API's resourceLinks field returns direct download URLs for attachments. When you append your public API key to those URLs, many public attachments can be downloaded without logging in through the browser. The login wall is a UI restriction, not an API restriction, for public (non-sensitive) attachments.

Rate Limits & Practical Constraints

  • Public API key: ~1,000 requests/day (non-federal); federal system accounts get higher limits
  • Practical rate limit: ~20 req/min before 429 errors (lower than documented 60/min)
  • System accounts require 10-business-day approval for full access to sensitive fields
  • API keys must be rotated every 90 days per ToS
  • The uiLink field returns direct opportunity URLs, but redirects to 404 unless logged in — do not use for automation

4. Legal & Compliance Considerations

Changeis Context: As a federal contractor holding active government contracts, Changeis faces a higher compliance bar than a typical commercial entity. Violations of SAM.gov ToS are not merely civil matters — they can trigger contract suspension, debarment proceedings, and CFAA exposure. The risk calculus is asymmetric.

SAM.gov ToS — Verbatim Prohibitions

"Automated data gathering, web scraping tools are prohibited and, if detected, will result in the associated account(s) being denied access to SAM.gov via Login.gov."

"Do not use your SAM.gov login for data mining, bots, or other data gathering and extraction tools."

"Systematic access (electronic harvesting) or extraction of content from the website, including the use of 'bots' or 'spiders,' is prohibited."

"Unauthorized attempts to upload or change information on this service are strictly prohibited and may be punishable under the Computer Fraud and Abuse Act of 1986 and the National Information Infrastructure Protection Act."

Source: SAM.gov Terms of Use

Allowed vs. Forbidden — Clear Lines

✓ Allowed

  • Use the official Opportunities API v2 with your API key
  • Download opportunity listings and metadata in bulk via Data Services
  • Access resourceLinks attachment URLs via API (public attachments)
  • Use USASpending.gov and FPDS-NG APIs freely
  • Use third-party platforms (HigherGov, GovWin) that have their own data agreements
  • Manually browse SAM.gov while logged in as a human user
  • Share public API data externally (non-sensitive fields only)
  • Use Browser Use for non-federal commercial sites with no ToS prohibition

✗ Forbidden (Changeis Specifically)

  • Running Browser Use, Skyvern, or any bot against SAM.gov's authenticated UI
  • Using SAM.gov Login.gov credentials in any automated script
  • Automated bulk download of attachment files via browser automation
  • Using browser agents to bypass CAPTCHA on SAM.gov
  • Sharing sensitive API data (FOUO, Entity Management FOUO) outside the organization
  • Sharing individual API keys with contractors without system account setup
  • Any "electronic harvesting" of SAM.gov content, automated or manual-mimicking

CFAA Risk for Federal Contractors

The Computer Fraud and Abuse Act (18 U.S.C. § 1030) criminalizes "exceeding authorized access" to a protected computer — and federal government IT systems unambiguously qualify. Courts have interpreted CFAA broadly for authenticated systems: even if a contractor has a valid SAM.gov account, using it in violation of ToS (e.g., bot-driven access) can constitute unauthorized access. Post-Van Buren v. United States (2021), the Supreme Court narrowed CFAA for publicly accessible data, but SAM.gov's login-gated attachment area is not publicly accessible — it requires authentication, putting it squarely in high-risk territory.

Additional risk layers for Changeis: A CFAA violation or federal ToS breach by a government contractor can trigger False Claims Act scrutiny, suspension and debarment proceedings under FAR 9.4, and reputational damage with agency clients. The asymmetry is severe: the efficiency gain from automating SAM.gov downloads is minimal given the official API provides attachment URLs anyway.

6. Recommended Path for Changeis

Primary Recommendation: Use the official SAM.gov Opportunities API v2 for all opportunity discovery and attachment access. Supplement with HigherGov or GovWin for intelligence overlays. Reserve Browser Use / browser automation tools exclusively for non-federal-portal workflows where ToS permits.

SAM.gov: Do This Instead

  1. Register for a SAM.gov public API key at sam.gov/profile/details — takes 1–5 business days, free.
  2. Query api.sam.gov/opportunities/v2/search with NAICS codes, keywords, agency, set-aside type. Returns full opportunity metadata.
  3. Extract resourceLinks from each opportunity record — these are direct download URLs for posted attachments. Append &api_key=YOUR_KEY to download public attachments programmatically, no browser required.
  4. For awarded contract research, use USASpending.gov API (api.usaspending.gov/api/v2/search/spending_by_award/) — no API key, no rate-limit drama.
  5. Implement exponential backoff — SAM.gov's real-world rate limit is ~20 req/min despite documented 60/min; plan for 429 retries.
  6. Request a System Account for higher limits and access to sensitive entity fields when Changeis needs vendor registration data.

When Browser Use IS Appropriate for Changeis

Use browser automation tools (Browser Use, Skyvern, Stagehand) only for:

  • Commercial intelligence platforms — HigherGov, GovWin, GovTribe portals (verify each platform's ToS; most permit programmatic access for subscribers)
  • Partner portals — Teaming partner intranets, Prime contractor portals, vendor onboarding portals that have no API
  • Internal tools — Costpoint, Unanet, internal SharePoint workflows where Changeis owns the system
  • Non-federal commercial sites — LinkedIn research, competitor website monitoring, market intelligence
  • Document assembly automation — Auto-populating proposal templates, price-to-win models from commercial data
