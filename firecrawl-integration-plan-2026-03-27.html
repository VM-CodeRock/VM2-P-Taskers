Key Gap
The current brief identifies opportunities accurately via API but shows no description text from the actual SAM.gov opportunity page. Every "Request Deep Dive" click currently triggers a separate manual investigation step. Firecrawl can fill this gap by extracting full description text at brief-generation time — enabling richer summaries and better RAG matching without human intervention.

SECTION 2
Firecrawl Integration Plan for SAM.gov / APFS

Firecrawl targets two specific gaps: extracting opportunity descriptions at scale from SAM.gov detail pages, and reliably parsing the JavaScript-heavy APFS forecast table. Below is the integration architecture.

Use Case 1 — SAM.gov Opportunity Description Extraction

HIGHEST VALUE Description Excerpts in Every Brief Card

SAM.gov's Public API returns a description field that is a link to the description, not the text itself. To get the actual scope text, you must fetch https://api.sam.gov/prod/opportunities/v2/{noticeId}/description — or load the detail page. Firecrawl's /scrape endpoint handles this with a Pydantic schema to extract structured opportunity data from that API endpoint, then injects a 2–3 sentence excerpt into each brief card. STRONG-match cards get full description + attachment links.

Python — SAM.gov Description Extraction via Firecrawl
# pip install firecrawl-py pydantic from firecrawl import FirecrawlApp from pydantic import BaseModel from typing import Optional, List import os app = FirecrawlApp(api_key=os.environ["FIRECRAWL_API_KEY"]) # Pydantic schema for structured opportunity data class OpportunitySchema(BaseModel): title: str description_excerpt: str # 2-3 sentence summary of scope set_aside_type: Optional[str] # e.g. "Small Business Set-Aside" place_of_performance: Optional[str] estimated_value: Optional[str] attachments: Optional[List[str]] # SOW/PWS/RFP download links contact_name: Optional[str] contact_email: Optional[str] def get_opportunity_detail(notice_id: str, sam_url: str) -> dict: """Extract structured data from a SAM.gov opportunity page.""" result = app.scrape_url( url=sam_url, params={ "formats": ["extract"], "extract": { "schema": OpportunitySchema.model_json_schema(), "prompt": ( "Extract the key procurement details from this SAM.gov " "opportunity page. For description_excerpt, write a concise " "2-sentence summary of what is being procured and the primary " "scope of work. Include all document attachment URLs." ) } } ) return result["extract"] # Usage: only run for STRONG-match opportunities to minimize credit use for opp in strong_matches: detail = get_opportunity_detail(opp["noticeId"], opp["uiLink"]) opp["description_excerpt"] = detail["description_excerpt"] opp["attachments"] = detail["attachments"] or []

Use Case 2 — APFS JavaScript Table Reliable Extraction

JS RENDERING APFS apfs-cloud.dhs.gov/forecast/ — JavaScript-rendered table

The APFS public forecast page at apfs-cloud.dhs.gov/forecast/ renders its data via JavaScript. Standard HTML parsing may see an empty table body (which is what a direct fetch returns). Firecrawl's /scrape endpoint runs a real browser engine (Fire-Engine) that waits for JS execution, ensuring all rows are captured. This replaces any brittle HTML parsing with a reliable structured extraction.

Python — APFS Forecast Table Extraction via Firecrawl
from pydantic import BaseModel from typing import List, Optional class APFSRecord(BaseModel): apfs_number: str component: str requirements_title: str naics: Optional[str] dollar_range: Optional[str] contract_vehicle: Optional[str] small_business_program: Optional[str] anticipated_award_date: Optional[str] estimated_solicitation_release: Optional[str] period_of_performance_start: Optional[str] period_of_performance_end: Optional[str] place_of_performance_state: Optional[str] contract_type: Optional[str] primary_contact_email: Optional[str] class APFSForecast(BaseModel): records: List[APFSRecord] total_count: int def fetch_apfs_forecast() -> List[dict]: """Fetch all APFS forecast records via Firecrawl (handles JS rendering).""" result = app.scrape_url( url="https://apfs-cloud.dhs.gov/forecast/", params={ "formats": ["extract"], "actions": [ {"type": "wait", "milliseconds": 3000} # wait for JS table render ], "extract": { "schema": APFSForecast.model_json_schema(), "prompt": ( "Extract all rows from the APFS Acquisition Forecast table. " "Capture every record with all available fields. Return " "total_count as the number of records found." ) } } ) return result["extract"]["records"] # Compare against yesterday's cached results to find new/updated records def find_new_apfs(today: List[dict], yesterday: List[dict]) -> dict: yesterday_ids = {r["apfs_number"] for r in yesterday} new_records = [r for r in today if r["apfs_number"] not in yesterday_ids] return {"new": new_records, "total": len(today)}

Use Case 3 — Deep Dive Auto-Enrichment (STRONG Matches)

AUTOMATION Auto-Populate Deep Dive Briefs via Firecrawl /agent

When a "Request Deep Dive" is triggered in Todoist, Firecrawl's /agent endpoint can autonomously gather all supporting data — FPDS incumbent data, USASpending agency spend, and the full SAM.gov opportunity text — in a single agentic call. This replaces the current manual researcher step with an automated 2–3 minute enrichment run.

Python — Deep Dive Auto-Enrichment
# /agent endpoint: describe the goal in plain English def auto_deep_dive(solicitation_number: str, title: str, agency: str, naics: str) -> dict: result = app.agent( prompt=f""" Research this federal procurement opportunity for Changeis Inc: Solicitation: {solicitation_number} Title: {title} Agency: {agency} NAICS: {naics} Gather: 1. Full opportunity description from sam.gov/opp search 2. Current incumbent contractor from fpds.gov (search by solicitation number) 3. Agency spending in NAICS {naics} from usaspending.gov (last 3 years, top 5 contractors) 4. Any related upcoming DHS opportunities from apfs-cloud.dhs.gov/forecast/ Return all findings as structured data. """ ) return result["data"]

SECTION 4
Claude Code + Firecrawl MCP Setup

Firecrawl has an official MCP (Model Context Protocol) server that connects it directly to Claude Code. Once set up, any Claude Code session can fetch, extract, and search live web content — including SAM.gov pages — in real time during a conversation. This is separate from the cron pipeline: it's for on-demand research inside Claude Code sessions.

What MCP gives you in Claude Code
Once the Firecrawl MCP server is installed, Claude Code gains the ability to: fetch and read any live webpage in the session, search the web and return full page content (not just snippets), crawl multi-page sites, and extract structured data on demand — all within the same Claude Code conversation. You can simply ask "go read this SAM.gov page" and it works.

Prerequisites (what you need before starting)

  • Node.js installed on your Mac — Claude Code requires Node.js. Check by opening Terminal and typing node --version. If you see a version number (e.g., v20.x.x), you're good. If not, download from nodejs.org — click the LTS button.
  • Claude Code installed — If you're using Claude Code, this is already done. If you need to install: npm install -g @anthropic-ai/claude-code
  • Firecrawl API key — Same key from Section 3. Free tier works for MCP testing.

Step-by-Step Setup (Claude Code MCP)

  1. Open Terminal on your Mac
    Press Cmd + Space, type "Terminal", press Enter. You'll see a black or white window with a command prompt.
  2. Run the MCP install command
    Type this command exactly (paste it in, then press Enter):
    claude mcp add --transport stdio firecrawl -- env FIRECRAWL_API_KEY=fc-YOUR-KEY-HERE npx -y firecrawl-mcp
    Replace fc-YOUR-KEY-HERE with your actual Firecrawl API key. This tells Claude Code where to find Firecrawl and how to authenticate.
  3. Verify it was added
    Run: claude mcp list
    You should see firecrawl appear in the list. If it does, setup is complete.
  4. Start a Claude Code session and test
    Open a Claude Code session in any project folder. Type: /mcp inside the session to see connected tools. You should see Firecrawl listed. Then try: "Use Firecrawl to fetch the current SAM.gov contract opportunities page and summarize what's there."

Alternative: Add to claude_desktop_config.json (if using Claude Desktop app)

// File location: ~/Library/Application Support/Claude/claude_desktop_config.json { "mcpServers": { "mcp-server-firecrawl": { "command": "npx", "args": ["-y", "firecrawl-mcp"], "env": { "FIRECRAWL_API_KEY": "fc-YOUR-API-KEY-HERE", "FIRECRAWL_CREDIT_WARNING_THRESHOLD": "500", "FIRECRAWL_CREDIT_CRITICAL_THRESHOLD": "100" } } } }

To open this file on a Mac: in Finder, press Cmd + Shift + G, paste ~/Library/Application Support/Claude/, and press Enter. If the file doesn't exist yet, create it with those contents.
