1. Executive Summary

Bottom Line: The coreyhaines31/marketingskills repository is a high-quality, MIT-licensed collection of 40 markdown-based AI agent skill modules covering CRO, copywriting, SEO, cold outreach, RevOps, and sales enablement. Built for Claude Code, Cursor, and OpenAI Codex, the modules follow the Agent Skills spec and are immediately installable via npx skills. For Changeis BD, 5 modules are directly adoptable with minimal adaptation for federal-context tasks; 8 additional modules are adaptable with effort. The repo is active, well-structured, and safe to fork under MIT.

Changeis operates in the federal professional services market — a context that differs substantially from SaaS (the repo's primary target). However, the underlying workflows for cold outreach, competitive profiling, sales enablement collateral, content strategy, and customer research are structurally identical regardless of sector. The primary adaptation required is swapping SaaS-native terminology and platform references (GA4, HubSpot, A/B testing) for federal-context equivalents (SAM.gov, GSA eBuy, capability statements, NAICS/CAGE codes, win-theme frameworks).

Recommended action: Fork the repo, adapt 5 priority modules for Changeis BD use cases, and integrate via Perplexity Computer skill scope=user as markdown context files. Full Claude Code integration is optional but not required in Phase 1.

2.2 Architecture: How Skills Work Together

The repo follows a hub-and-spoke dependency model. The product-marketing skill is the universal context document — every other skill reads it first to understand product, ICP, and positioning before executing tasks. This maps well to Changeis: replacing the SaaS product context with a Changeis BD context document (NAICS codes, core capabilities, past performance, contract vehicles) would feed all downstream skills automatically.

Key architectural insight: The product-marketing skill creates a .agents/product-marketing.md file. For Changeis, this becomes a BD context document containing: company overview, core capabilities, NAICS/CAGE/UEI, active contract vehicles (OASIS+, CIOSP4, SEWP), key customers (FAA, DHS, DoD), win themes, and competitive differentiators. All other adapted skills then reference this automatically.

2.3 Skill Format Deep Dive

Each SKILL.md file contains:

  • YAML front-matter: name, description (trigger phrases for agent routing), metadata.version
  • Role/goal statement: Defines agent persona and objective
  • "Before Starting" block: Always reads product-marketing context first, then asks clarifying questions
  • Workflow steps: Numbered, actionable, format-specific (tables, bullet lists, email templates)
  • Quality criteria: Specific rubrics (e.g., "subject line under 50 chars," "one CTA per email")
  • Related skills cross-references: Dependency map for chaining

The format is directly portable to Perplexity Computer skill scope=user — the YAML front-matter is ignored by Perplexity's skill loader (which uses plain markdown), but the workflow content is fully usable as-is.
