Executive Summary

On March 30, 2026, Microsoft launched Critique and Council — two multi-model intelligence capabilities inside the Microsoft 365 Copilot Researcher agent — that fundamentally change how AI-generated research and documents can be verified before delivery. Critique uses a two-model sequential pipeline (one model generates, a second independently reviews and refines), while Council runs OpenAI and Anthropic models in parallel with a judge model to surface where they agree and diverge. Both are available through the Microsoft 365 Frontier program for enterprise customers.

Critique's architecture directly targets these failure modes: its rubric-based reviewer enforces source reliability, report completeness, and strict evidence grounding, producing a 13.88% accuracy improvement over prior best-in-class deep research benchmarks.

Copilot Critique Overview

What is Critique? Critique is a new multi-model deep research architecture within the Microsoft 365 Copilot Researcher agent, announced March 30, 2026. Unlike traditional single-model AI workflows — where one model handles planning, sourcing, synthesis, and writing — Critique divides these responsibilities between two specialized AI partners drawn from frontier labs including OpenAI and Anthropic.

Model 1 — Generator

Exploration & Synthesis

  • Plans the research task and scope
  • Iterates through retrieval and source selection
  • Produces a structured initial draft with citations
  • Optimized for depth, breadth, and analytical framing
Generation Phase
Model 2 — Reviewer

Validation & Refinement

  • Acts as an independent expert reviewer
  • Enforces source reliability and citation integrity
  • Checks completeness vs. original request intent
  • Strengthens evidence grounding; flags unsupported claims
Review Phase

How it works — the three review dimensions: The reviewer model applies a rubric-based evaluation similar to academic peer review across three explicit dimensions:

Review Dimension What is Checked Federal Document Relevance
Source Reliability Assessment Prioritizes reputable, authoritative, domain-appropriate sources; deprioritizes weak signals Critical for regulatory citations (FAR, DFARS, CMMC), ensuring only official sources are cited in compliance documents
Report Completeness Evaluates whether the report satisfies the original request intent comprehensively with unique, relevant insights Maps directly to Section L/M compliance in federal proposals — every requirement must be addressed
Strict Evidence Grounding Every key claim must be anchored to a reliable source with precise citations; removes unsupported assertions Audit trail integrity for CDRL deliverables, audit responses, and compliance reports requiring traceable claims

What is Council? Council is a parallel alternative mode: both an Anthropic model and an OpenAI model independently produce complete reports on the same query, then a dedicated judge model synthesizes consensus, flags divergences, and surfaces unique contributions from each. Council costs approximately 2.5× a single-model query and is best reserved for high-stakes decisions where conflicting analytical framings matter — such as strategic proposal narratives or competitive positioning documents.

Availability Status
Both Critique and Council are broadly available through the Microsoft 365 Frontier program as of March 30, 2026. Critique is the default experience in Researcher when "Auto" is selected in the model picker. Council is available by selecting "Model Council." The M365 E7 Frontier Suite ($99/user/month, launching May 1, 2026) bundles E5 + Copilot + Entra Suite + Agent 365 for organizations seeking a comprehensive package.

Security Architecture
Data sovereignty: All GCC High Copilot interactions are processed within the tenant boundary. Model providers (OpenAI, Anthropic, Google) cannot train on enterprise data. Microsoft Entra ID enforces role-based access. Microsoft Purview provides audit logging for all AI interactions — creating the compliance trail required for federal document production workflows. Web grounding is OFF by default, ensuring sensitive government content does not leave the GCC High boundary.

GSA OneGov pricing: Federal agencies and qualifying contractors purchasing through the GSA under the OneGov agreement can access Microsoft 365 Copilot at no additional cost for up to 12 months with an M365 G5 license. This eliminates the typical $30/user/month Copilot add-on cost during the pilot period — a material incentive to move quickly. Security Architecture

Changeis operates under federal contracting obligations that require strict data handling. The Copilot GCC High architecture addresses these requirements by design, as documented in the Microsoft GCC High announcement and Nextgov/FCW reporting:

  • FedRAMP High authorization — Copilot in GCC High operates within the FedRAMP High authorization boundary, meeting the highest federal security baseline
  • DFARS / ITAR / CMMC compliance — Explicitly designed for DIB contractors; supports CMMC 2.0 Level 2 and Level 3 environments
  • Data residency — All data stored and processed in U.S.-based Microsoft data centers operated by screened U.S. personnel
  • Tenant boundary enforcement — Model providers (OpenAI, Anthropic) cannot access or train on tenant data; all processing governed by Microsoft's enterprise data protection
  • Web grounding OFF by default — Prevents inadvertent exposure of sensitive proposal content to external web services; can be selectively enabled for permitted use cases
  • Microsoft Purview integration — Full audit logging of all Copilot interactions (including Critique reviewer outputs); supports eDiscovery, DLP, and compliance monitoring
  • Microsoft Entra ID (Azure AD) — Role-based access control for all Copilot features; integrates with existing Changeis identity governance
  • Copilot Control System — Centralized admin console for license management, usage monitoring, and policy enforcement — critical for managing GFE restrictions and need-to-know access
Key Risk Mitigation
The primary security risk for AI-assisted document production is inadvertent data leakage to model training pipelines. The GCC High architecture eliminates this risk through contractual and technical controls: Microsoft's enterprise data protection commits prevent model training on customer data, and the FedRAMP High boundary ensures all processing stays within compliant infrastructure. Proposal content, CDRL text, and compliance data can be used as Copilot inputs without violating CUI or contract data handling requirements.
